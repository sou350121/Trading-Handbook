# 投資哲學量化：一份五案全結的預註冊研究計畫（實戰紀錄）

> **這一頁不是文獻。** 是一位個人量化交易者在 2026-07 用預註冊協定，把五套著名投資哲學逐一放上檢驗台的第一手紀錄——Buffett 的跟單、Dalio 的全天候、Greenblatt 的魔法公式、風險平衡的加法分散、以及「保本」哲理的低波動載具。**五案全部結案，無一晉級部署。**
>
> 手冊裡的其他頁面問「這篇 paper 的 claim 在真實市場會不會崩」。這一頁提供的是反方向的證據：**當一個人真的按預註冊紀律去測，那些最著名的投資哲學崩在哪裡、怎麼崩。**

**Status:** v0.6 — 一手紀錄，Opus 綜合。所有數字逐字引自原始裁決文件；未計算過的一律標「未評估」。與 [自動化搜索 vs 預註冊驗證](/crossing/autoresearch-vs-preregistration) 是同一組證據的兩面：那一頁講文獻側沒有 trial 記帳，這一頁講實戰側**有**記帳時會發生什麼。

## 為什麼這頁值得存在

在 [自動化搜索 vs 預註冊驗證](/crossing/autoresearch-vs-preregistration) 裡有一個經三票對抗驗證的結論：整條自動因子挖掘文獻（GP / RL / LLM agent）沒有任何一篇實施多重檢定校正，也沒有任何一套系統公開過「預註冊 + 明確 trial count + deflated Sharpe/PBO」齊備的樣本外紀錄。

這一頁是那個空白的一個對照樣本。它的樣本數小（5 案、數十個變體）、宇宙窄（美股 ETF 為主 + A 股 / 13F 個股），**但它的門柱是在計算之前用 git commit 釘死的**，時間戳可查。這是文獻側普遍缺席的那一半。

**它最有價值的地方是全部結果都是負的。** 負結果幾乎不會被發表，於是每個人都以為這些哲學可行——因為看不到失敗的那些。

## 協定：四層框架 + 門柱先於計算

**四層框架**（每個候選都要逐層過）：

| 層 | 問什麼 |
|---|---|
| **標的層** | 金融特性、流動性、價格相關性——這個東西本身是什麼？ |
| **組合層** | 資訊相關性 / 邏輯相關性 / 效果相關性——它跟我已有的東西是不是同一筆交易？ |
| **回測層** | 真實數據 + 仿真數據——它在歷史上做了什麼？ |
| **Gap 層** | 預期 vs 實際——我事前說它會怎樣，它實際怎樣？ |

> 組合層的三種相關性是這套框架最不直觀也最有用的部分：**統計相關性低不等於獨立**。兩個訊號可以月相關 0.2 但吃同一個資訊源、靠同一個邏輯賺錢——那它們在危機時會一起死。詳見 [Crossing · 自動化搜索 vs 預註冊驗證](/crossing/autoresearch-vs-preregistration) 的「效果相關性」討論。

**流程紀律**（這是全部價值的來源）：

1. 所有閘門（gates）在**任何計算開始之前**寫成文件並 git commit——時間戳為證。
2. 執行者（worker agent）只負責填數字，不參與判斷。
3. 裁決逐關對照，**門柱不可移**。
4. 自製版對照組**盲於**被對照方的結果註冊。
5. 最後一道：**對抗性裁判**——一個獨立 agent 攻擊已經寫好的裁決推理本身。

第 5 條抓到過實質錯誤（過度措辭、編造對照組），是後來固化進標準流程的。

## 五案

### 案 1 · Buffett / Berkshire 13F 跟單 — 負（但翻案兩次才定案）

**哲學主張**：跟著最好的價值投資者買，你不需要自己選股。

**引擎**：真 point-in-time EDGAR 13F，110 季（1998Q4–2026Q1），AAPL 正確首現 2016Q1（驗證 PIT 無前視）。

**四輪推進後，細審抓到最大的一條魚——現金假象。** 回測引擎把「未定價 / 未映射 / 不新鮮」的名稱默默留作 0%，等於當現金。結果早期組合日均隱性現金 **45–55%**。修正成全投資後，**兩個已發表的結論被撤回**：

| 原結論 | 修正後 |
|---|---|
| 含 AAPL 全樣本也輸 SPY | **反轉**——滿倉勝 SPY **+1.5 / +1.7pp**（ex-AAPL 仍 +1.3 / +1.4） |
| GFC 保護 +14.7pp | **+1.2pp**——原本那 14.7pp 幾乎全是現金的功勞 |

**這一條值得所有人記住：隱性現金會同時偽造「保本能力」與「跑輸大盤」兩個相反的結論。** 本案兩個結論同時中招。

修正後的逐危機保護：**+24.8 / +1.2 / −5.6 / +1.0 pp**。真正的保護只發生過一次（dot-com），而那次的原因是 2000 年的持股裡**恰好零科技**——是組合的反泡沫構成，不是經理的擇時能力。COVID 那次滿倉比 SPY 更差。

**裁決：不部署。** 理由不是「賺得少」，是**優勢體制集中**：滾動起點 Sharpe 只贏 1/6，2004 年後遞增地輸，2010 年後連 MDD 都輸。裁判特別修正措辭——說「**體制集中、平穩性未確立**」，不說「起點假象」。

**副產品（比主結論更有用）**：低 beta 長倉的下檔緩解在長史上持久（MDD 勝 10/10，bootstrap 95.3%），但 Sharpe 只勝 6/10 = **報酬效率未確立**。可交易主張因此收窄為「長倉低 beta 的下檔緩解歷史持久」，而不是「低波動異象可以賺錢」。

### 案 2 · Dalio 全天候 — 三連測，三連殺

三份預註冊文件，三個獨立標的：一篇聲稱「十年 Sharpe 1.14 / MDD < 4%」的中國版全天候文章、一套自製 CN 四引擎、一套自製 US 四引擎。**三個都不晉級。**

**(a) 文章版：作者數學誠實，但可交易化殺死它。** 復現 PASS——CAGR 6.38% vs 聲稱 6.3%。死在四個地方：真 ETF 版 Sharpe 只剩指數版的 **0.60×**（主因是可交易債券 ETF 缺全債指數的信用 carry）；逆波動「動態」配置的 C/M 1.081 **輸給它自己的凍結配比 1.137**（風險平價在這裡是裝飾）；2016Q4 債災回撤 4.76% > 註冊的 4% 門檻；**前瞻 E[r] 1.77% vs 貨幣基金 1.4%**。

> 修正了一個我原本的預測：這十年報酬**不是**利率下行的資本利得（僅占債腿 11.6%），**是已經消失的 3–4% carry 時代**。所以它是時代依賴，不是過擬合——這個區分很重要，因為兩者的補救方式完全不同。

**(b) 自製 CN 四引擎：結構陷阱。** 輸給文章版（毫釐之差，但門柱不移）。真正的發現是：**無槓桿的波動平衡在 1.7% 收益率的債市必然塌縮成債券基金**——ERC 把 66% 的資本推進債券，前瞻久期占比 90.5%。**「CN 全天候」這個概念整體不值得做，與具體機制無關。**

**(c) 自製 US 四引擎：最接近成功的一個，仍不晉級。** 乾淨贏 naive（C/M 0.498 vs 0.349）、危機 8/8 全過、**2022 全年 −2.62% vs naive −7.27%**（趨勢閘確實交付）、小額帳戶的整數股拖累只有 0.049pp/年（可執行性成立）。但 **滾動起點 4/7 FAIL**——順風年裡趨勢閘的鞭打費 + 現金拖累讓它輸。依預註冊，不晉級。

**而且不做 v2。** 消融顯示要砍的部件修不了滾動起點那關；且它對既有趨勢 sleeve 相關 0.468、**水下日重疊 98.5%**——它是既有 sleeve 的表親，搶同一個預算位。**迭代到通過 = 元過擬合。**

**跨市場機制知識（本案真正的產出）**：趨勢閘的價值前提是**主導引擎會趨勢性崩盤**。US 2022 大賺，CN 債券主導的十年沒有破位、只付鞭打費。所以「持久性屬於演算法而非資產」要精確化成——**屬於與體制結構匹配的演算法**。

另外：**可轉債在中美都不是獨立引擎**（對股權相關 0.70 / 0.80，拿掉後 C/M 反而上升）。把 1.5 個引擎數成 2 個，是消融義務抓出來的自己的設計錯誤。

### 案 3 · Greenblatt 魔法公式 — 負（在有利偏差下全敗）

**引擎**：SEC `companyfacts.zip`（1.39 GB / 20,068 家 / 含已下市公司 / filed 日期齊全），四步親自驗真，AAPL 對線上 API 逐位一致。

**根本數據缺口，誠實標記**：免費數據沒有歷史 security master（SEC 的 ticker map 是當期快照），所以真實的歷史 top-750 宇宙**不可識別** → 形式上 **NOT-EVALUABLE**。執行的 agent 標記了這一點而沒有冒充可評估——這一步本身是流程的勝利。

**但 pilot 的倖存者缺口是偏向策略的**（缺的是死掉的公司），即便在這個有利偏差下仍然全面失敗：

| 指標 | 魔法公式 | SPY |
|---|--:|--:|
| C/M | 0.324 | **0.417** |
| Sharpe | 0.683 | **0.854** |
| 滾動起點 | **0/7 全敗** | — |
| 對照三檔因子 ETF（VLUE/QUAL/COWZ） | **全輸** | — |
| 換手 | 127% / 年 | — |

**有利偏差下的全敗 = 上界。** 所以不值得再花錢買 security master 重跑。**結論：在現代體制（2010–2026）魔法公式不成立。**

**最有價值的產出是「靈魂關」——這個書跟我已有的東西是不是同一筆交易？** 答案出乎意料：小市值價值書在**組成上真的是不同的交易**——回撤重疊 Jaccard **0.350**、月相關 0.622、對趨勢 sleeve **−0.10**（完全正交）。「delta-one 全是同一筆交易」這個先驗在組成層被部分反駁了。

**但 126 日滾動相關的 p90 是 0.818**——壓力期耦合回歸。

合起來：**「不同的交易、更差的交易」。平時分化是真的，危機時（也就是正好需要分散的時候）耦合，而且沒有報酬 edge。** 這是「危機 ρ 天花板」的第四個獨立證據。

### 案 4 · 風險平衡的加法分散 — 阻尼器，不是引擎

Dalio 式「加一個低相關資產進去」的主張，在四層框架下被降級：加進來的東西與主書的滾動 p90 相關 0.86–0.89 = **阻尼器**——它降低波動，但它跟主書的防禦 overlay **搶同一個預算位**，而不是開一條新的收益來源。

### 案 5 · 「保本」哲理 → 低波動 sleeve — 兩個 host 都不晉級

用同一套四層框架把「保本」這個哲理的存活候選（USMV 類低波動 sleeve）跑完。

- 兩個 host（兩套不同的主書）**都不晉級**。
- **最有資訊量的一關**：20% 短債（BIL）的 C/M（0.455 / 0.551）**勝過** 20% USMV（0.437 / 0.504），兩個 host 都成立。對一個已經自帶防禦的系統，**國庫券是更好的邊際保本撥盤**。
- 角色釘死：USMV vs host 滾動 p90 相關 0.86–0.89 = 阻尼器；USMV vs 趨勢 sleeve ≈ 0（COVID −0.28）= **趨勢 sleeve 仍是唯一效果正交的 sleeve**。
- 長史關 3/3 過：低 beta 的保本邏輯在因子層是持久的——**哲理對，但這套系統裡可部署的內容已經被既有的防禦 overlay + 趨勢 sleeve 吃掉了**。

**最誠實的一筆記錄**：holdout（2024+）方向強 PASS（+3.5 ~ +3.9pp）——**這正是預註冊要防的那種近期誘惑**。記成觀察項，不改判。

## 這五案教了什麼（合成）

**1. 保護是「組成 / 演算法」的性質，不是「跟單某人」的性質。** 三次獨立驗證：同一個經理四次危機，只有持股恰好反泡沫的那一次有保護；而平穩的低 beta 因子四次全部緩衝。**你買的應該是規則，不是人。**

**2. 危機 ρ 天花板：delta-one 分散有上限，而且四次獨立撞到。** 相關性拆解、凸性對沖、全天候四引擎、魔法公式靈魂關——四條完全不同的路徑，全部撞到同一堵牆：**平時看起來正交的東西，在壓力期 p90 相關會回到 0.8 以上。** 這不是選錯資產，這是 delta-one 這個類別的結構天花板。要突破必須換工具（凸性 / 趨勢 / 現金），不是換標的。

**3. 「持久性屬於演算法而非資產」要再精確一格：屬於與體制結構匹配的演算法。** 趨勢閘在會趨勢崩盤的市場交付，在沒有破位的市場只付鞭打費。同一個演算法，兩種命運。

**4. 隱性現金會同時偽造兩個相反的結論。** 這是最容易中招也最難察覺的引擎 bug，因此升格成一道通用閘門：**投入率審計**。

**5. 滾動起點是單一回測期最好的替代品。** 五案裡有三案是死在滾動起點上的（13F 1/6、Greenblatt 0/7、US 4E 4/7），而這三案在單一固定窗上都看起來不錯甚至很好。**這正是文獻側普遍缺失的那一關**——AlphaGen 用 2020-2021 一個固定窗、AlphaQCM 用 2021-2022 一個固定窗，兩者都不會照出這個東西。

**6. 全部負結果的價值：確定性。** 五輪對照下來，既有系統一個字都沒改。這本身就是資訊——**這輪買到的不是收益，是確定性。**

## 對接自動化：為什麼這套紀律不能直接餵給 loop

這是本頁與 [自動化搜索 vs 預註冊驗證](/crossing/autoresearch-vs-preregistration) 的接縫，也是把這套人工紀律自動化時**唯一真正危險**的地方。

上述五案的 trial count 是**手工量級**——數十個變體，人可以誠實地在心裡記帳。門柱釘死之所以有效，是因為 N 小到「通過六關」這件事本身還帶有真實資訊。

**把 N 推到 10³–10⁵，同一套閘門會被暴力穿透，而且穿透時看起來完全正常。** Bailey / López de Prado 的隨機漫步示範就是這個過程的證明：8,800 個組合裡挑出來的贏家，PSR 說「真 Sharpe 為負的機率 < 1%」，而它的真實 PBO 是 55%。**你的六關不會報錯，它會發一張漂亮的通過證書。**

更糟的是，最自然的補救——「把校正指標加進 fitness」——被 PBO 的原作者明文禁止（一旦成為目標就失效）。

所以從這五案到自動化研究迴圈，缺的不是算力，是**帳本**：

| 手工時代（本頁） | 自動化時代（需要補的） |
|---|---|
| 門柱先於計算，git commit 為證 | 同左，但門柱必須在迴圈**構不到**的一層 |
| N 小到可以心算 | **Trial ledger**——每個候選都登記，含被丟棄的 |
| 通過/不通過（boolean） | **條件於 N 的機率**（deflated Sharpe / PBO with effective N） |
| 人決定何時停止 | 前置 pretest（vs 隨機搜索 null）+ 停止規則 |
| 滾動起點 + 凍結 holdout | 同左，但 holdout 成為**不可再生資源**，每次使用都要記帳 |

具體怎麼接，見 [自動化研究迴圈建造指南](/guides/autoresearch-loop)。

## 誠實邊界

- **樣本小、宇宙窄。** 5 案、數十個變體、以美股 ETF 與 13F 個股為主。這裡沒有任何一條結論具備文獻級的普遍性。
- **全部是同一個研究者的判斷。** 對抗性裁判流程降低了單點偏誤，但沒有消除它。
- **負結果的 power 未量化。** 「不晉級」有多少是真的沒有效果、有多少是門柱太嚴導致的漏真，本計畫**未評估**。這是這套紀律自己的盲點，與文獻側的盲點正好互補。
- **未做多重性校正。** 諷刺的是，本計畫自己也沒有計算 deflated Sharpe 或 PBO——N 小是理由，不是藉口。這是它自己的待辦。
