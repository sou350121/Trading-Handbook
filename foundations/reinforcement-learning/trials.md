# TRIALS 解構

> **發布**：2024-11-10 · （無 venue）
> **QuantML 導讀**：[选择与交易：基于分层强化学习的统一配对交易](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247487603&idx=1&sn=948fc01570263aab80b86a1d6475f89c&chksm=ce7e776df909fe7b9c9776a42520a015415ebb27414feef47a757ac4dae4c78cd323653fe43c#rd)
> **核心定位**：落點於日频波段與組合執行優化，以分層強化學習解耦傳統配對交易中「選擇-執行」的資訊斷層，將統計套利從兩階段硬編碼轉為端到端聯合策略搜索。

**五軸座標**

| 數據模態 | 時間尺度 | 學習範式 | Alpha機制 | 人機協作 |
|:-:|:-:|:-:|:-:|:-:|
| `量价表格` | `日频波段` | `强化学习` | `组合执行优化` | `全自动黑盒` |

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 將配對選擇與交易執行統一為分層強化學習任務，經理層選對、工人層下單。② 核心 trick 在於 Bi-GRU 結合時間注意力編碼狀態，並用 A2C 聯合優化兩階段策略。③ 這對「組合執行優化」軸的關鍵在於打破預選資產對的過擬合，讓選擇與執行在信用分配上閉環。④ 導讀未給量化結果。

**X-Ray.** 放回五軸 Pareto，TRIALS 將配對交易從「統計檢驗+閾值觸發」的開環流程，推向「狀態編碼+聯合信用分配」的閉環搜索。它解了兩個舊工程坑：一是預選階段忽略執行成本與滑點導致的虛高勝率，二是單一交易代理在固定資產對上容易陷入局部極值。然而，其 envelope 打不開高頻微結構與跨市場資金流依賴；Bi-GRU 的序列長度與注意力計算在日频波段雖可接受，但狀態空間維度隨資產池擴大呈平方級膨脹，組合選擇的探索效率將成為瓶頸。對量化讀者而言，此框架證明了「選擇即執行的一部分」，但實戰需警惕分層策略的信用分配延遲與非平穩市場下的注意力漂移。

## §1 · 架構 / Core Mechanism
**1.1 三大改動 vs 前作**
| 維度 | 傳統兩階段法 | TRIALS | 改動本質 |
|---|---|---|---|
| 任務結構 | 解耦（先選對後交易） | 統一分層（經理選對+工人交易） | 消除資訊傳播阻斷 |
| 狀態編碼 | 靜態統計特徵/固定閾值 | Bi-GRU + 時間注意力 | 動態捕捉序列顯著性 |
| 優化目標 | 獨立最大化 | A2C 聯合信用分配 | 選擇與執行報酬閉環 |

**1.2 ⚡ Eureka 一句話 trick**
把「選哪對」當作上下文強盜問題的選項，讓低層交易回報直接反饋給高層選擇策略，實現跨時間尺度的信用分配。

**1.3 信息流 ASCII 圖**
```
Market Data (Open/Close/Volume) 
   → Bi-GRU + Temporal Attention → State Embedding
   → High-Level Manager (Pair Selection) → Selected Pair
   → Low-Level Worker (Execution) → Action (Long/Short/Cover)
   → Reward (Cumulative PnL) → A2C Update (Joint)
```

## §2 · 數學層
📌 **Napkin Formula:** 
狀態編碼 $h_t = \text{BiGRU}(x_t)$, 注意力權重 $a_t = \text{softmax}(\frac{q k^T}{\sqrt{d}})$, 策略分佈 $\pi = \text{softmax}(W h_t + b)$, A2C 梯度更新 $\theta \leftarrow \theta + \alpha \nabla \log \pi(a|s) A(s,a)$。複雜度隨序列長度與資產池組合數平方膨脹。
**直覺:** 高層用交易期累積利潤當外在獎勵，低層用形成期內利潤當內在獎勵，兩層共享注意力機制的狀態表示，避免長序列梯度消失。
**Loss/訓練細節:** 採用 A2C，高層基於交易期表現更新，低層基於形成期內在獎勵更新，聯合優化兩階段策略。

## §8 · For the Reader
- **因子研究員:** 關注時間注意力機制的特徵權重可解釋性，可嘗試將傳統價量因子替換為 Bi-GRU 隱狀態，觀察因子衰減曲線。
- **高頻執行:** 此框架為日频波段設計，直接降頻至分鐘級將面臨狀態空間維度災難與 A2C 收斂不穩，需重構時間抽象層。
- **組合配置:** 分層選擇機制可擴展至多資產權重優化，但需重寫獎勵函數以納入組合層面的風險預算與相關性矩陣。
- **RL 策略:** 警惕高層選項空間隨資產數平方膨脹，建議引入圖神經網絡或剪枝策略降低探索成本，並加入正則化項防止注意力過擬合。
