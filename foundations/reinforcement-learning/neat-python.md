# NEAT-Python 解構（NEAT-Python）

> **發布**：2025-01-31 · （無 venue）
> **QuantML 導讀**：[基于神经进化拓扑结构(NEAT)算法和多技术指标共振的股票交易策略](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247489039&idx=1&sn=effb64c9ad581a0ba67a466758041fa3&chksm=ce7e7111f909f80799c5dde6e626323a715d37a16e9d72bbab4a562209ad121556c8bbbaef98#rd)
> **核心定位**：落點於「元學習搜索」與「端到端表征」軸，以 NEAT 演化算法替代梯度下降，直接搜尋 RNN 拓撲與交易邏輯的聯合結構。解了傳統因子挖掘中「特徵工程與交易決策脫節」的 prior gap，讓適應度函數直接錨定交易行為學而非預測精度。

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 將 NEAT 神經進化算法應用於日頻波段交易，透過多目標適應度函數與漸進式訓練數據策略，自動搜尋 RNN 拓撲結構。② 核心 trick 在於適應度函數的迭代設計：從單一 SQN 轉向加入「獎勵交易次數、懲罰平均持倉時間」的複合指標，強制網路放棄買入持有，轉向波段操作。③ 對「元學習搜索」軸而言，它示範了如何用演化壓力直接錨定交易行為學，而非僅優化預測 MSE。④ 實證顯示其回報標準差較買入持有策略低20%，但導讀未給絕對收益/回撤等量化結果。

**X-Ray.** 本方法落在「元學習搜索」與「端到端表征」的交匯處，本質是用演化算法替代梯度下降來搜尋交易邏輯的拓撲結構。它解決了傳統因子挖掘中「特徵工程與交易決策脫節」的工程坑，讓網路直接從技術指標共振中學習開倉/平倉/倉位管理的聯合策略。然而，其 Pareto 前沿明顯偏向「低波動與低暴露」，代價是平均回報低於買入持有。預測其打不開的 envelope 在於：缺乏對交易成本與滑點的內生建模，且 NEAT 常見的冗餘節點問題會導致推理延遲與過擬合風險。對量化讀者而言，其價值不在於直接實盤，而在於提供了一套「用適應度函數錨定交易行為學」的範式，可與強化學習的 Reward Shaping 互補。

## §1 · 架構 / Core Mechanism
| 改動階段 | 前作/基線邏輯 | 本方法改動 | 工程意圖 |
|---|---|---|---|
| 適應度設計 v1 | 單一 SQN 指標 | 失敗：網路演化出「收盤買入並持有至結束」策略 | 暴露單一收益指標無法約束行為學 |
| 適應度設計 v2 | 多目標：PnL + vs_BH_PnL - MaxDD | 失敗：仍傾向買入持有，產能利用率過高 | 風險懲罰未觸及「暴露時間」核心 |
| 適應度設計 v3 | 加入 `Reward(TradeCount)` 與 `Penalty(AvgHoldTime)` | 成功：強制網路轉向短持倉波段交易 | 將演化壓力直接錨定至交易頻率與持倉週期 |

⚡ **Eureka:** 適應度函數加入「獎勵交易次數」與「懲罰平均持倉時間」，直接將演化壓力從「預測準確率」錨定至「交易行為學」，迫使網路放棄被動暴露，轉向主動波段。

## §2 · 數學層
📌 **Napkin Formula:**
`Fitness = PnL + PnL_vs_BH - λ₁·MaxDD + λ₂·TradeCount - λ₃·AvgHoldTime`
*(導讀未披露 λ 權重，僅定性描述三項指標的加減方向)*

**直覺:** 多目標優化強制網路在追求收益的同時，必須通過「高頻換手」與「短持倉」來規避回撤。本質是行為約束（Behavioral Constraint），而非傳統的時間序列預測 loss。
**訓練細節:** 無梯度 backprop。依賴 `backtesting.py` 生成回測報告計算適應度。採用漸進數據策略：前1500代用90天數據（聚焦趨勢反轉與總回報），次400代用150天數據，末100代用1年數據（聚焦產能與風險管理）。

**解讀:** 該 Δ 反映的是波動率收斂與暴露時間縮短，非絕對 Alpha。導讀未提供 Sharpe/IR/MDD 等絕對指標，無法判斷是否覆蓋交易成本。漸進訓練可能引入數據泄漏（後期訓練含前期數據），且隨機切片回測未控制時間序列自相關，實盤有效性待驗證。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations:** 演化過程中產生大量未在決策中使用的節點、連接與輸入，這是 NEAT 算法的常見問題，會拖累推理效率並增加過擬合風險。
**6.2 推斷的隱含假設:**
- **Regime 依賴:** 策略依賴日頻波動率與趨勢反轉，若市場進入低波動/單邊趨勢 regime，短持倉邏輯將頻繁觸發假信號。
- **成本/滑點:** 完全未計入交易成本與滑點，高交易次數獎勵在實盤中會迅速侵蝕淨值。
- **數據泄漏/i.i.d. 假設:** 隨機抽樣訓練/回測假設數據獨立同分佈，違反金融時間序列的自相關與結構性斷點特性。
- **Survivorship:** 標普500歷史數據未提及剔除退市股，存在隱性 Survivorship Bias。

🎤 **Interview Tip:** 
- **正確答:** NEAT 的優勢在於自動搜尋網路結構與權重，能跳出局部極小值並直接優化交易行為指標（如持倉時間）；劣勢是評估成本高、易產生冗餘節點，且缺乏對成本內生建模。
- **錯答:** 把它當成普通的 LSTM/GRU 預測模型，忽略其「適應度函數錨定交易邏輯」的本質，或誤以為演化算法能自動解決過擬合。

**7.1 可證偽預測:** 若 2025-06-30 前未公開完整回測代碼與成本調整後的 Sharpe > 1.0 實證，則該方法僅具教學價值，無法進入生產環境。

## §8 · For the Reader
- **因子研究員:** 可借鏡其「多指標共振」輸入設計，但需將適應度函數改為對標因子 IC/IR，而非直接優化 PnL，以避免前瞻偏差。
- **組合配置/波段交易者:** 其「獎勵交易次數、懲罰長持倉」的邏輯可直接轉化為倉位管理規則，用於過濾低效暴露與控制最大回撤持續期。
- **RL/演化算法研究者:** NEAT 的漸進數據訓練（Curriculum Learning）是降低評估耗時的關鍵，可與 PPO/SAC 的 Curriculum RL 對比實驗，驗證行為約束的泛化性。
- **高頻執行:** 不適用。日頻切片與隨機抽樣完全無法捕捉微結構摩擦，且 NEAT 推理延遲不適合低延遲場景。
