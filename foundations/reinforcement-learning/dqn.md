# DQN 解構（DQN）

> **發布**：2025-04-12 · （無 venue）
> **QuantML 導讀**：[深度强化学习在多资产交易中的应用](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247490029&idx=1&sn=656e50ad82af1509e5b059670eb39129&chksm=ce7e7ef3f909f7e54356c728e2607c19c00c88a7430a890b1742dfa40ebcfff4926f0a565509#rd)
> **核心定位**：落點於「端到端表征 × 全自动黑盒」軸，試圖繞過傳統因子工程的特徵稀疏性與滯後性。解了 RL 在金融實盤中常見的「真實數據樣本不足導致探索崩潰」與「波動率 regime 切換使固定閾值失效」兩個 prior gap。

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 構建多資產期貨交易的 MDP 框架，以 DQN 驅動離散持倉決策。② 核心 trick 在於將多週期風險調整回報直接映射為狀態空間，並引入 GBM/VG 模擬數據擴充訓練集。③ 這對「端到端表征」軸具有標竿意義：代理直接從波動率標準化的價格序列中學習風險溢價，無需人工設計因子。④ 導讀未給量化結果。

**X-Ray.** 該方法在五軸 Pareto 前緣上，以「模擬數據依賴」換取「狀態空間的風險中性化」。傳統量價表格輸入 RL 常因異質波動率導致 Q-value 發散，此處用 EWMA 標準差進行跨週期縮放，實質是將狀態空間投影到風險調整後的流形上，大幅降低策略對波動率 regime 的敏感度。工程坑方面，它繞過了特徵選擇的組合爆炸，但將壓力轉移至模擬數據的生成保真度（GBM/VG 參數估計）與獎勵函數的稀疏性設計。預測其打不開的 envelope：離散動作空間 {-1, 0, 1} 與比例/固定成本模型脫節，無法處理滑點敏感或流動性分層的市場；且獎勵僅依賴價格變動乘積，未內建風險懲罰項，在尾部風險事件中易出現過度暴露。對量化讀者的意義在於驗證了「狀態風險標準化 + 模擬擴充」作為 RL 交易冷啟動的可行路徑，但實盤部署前必須補齊交易成本內生化與連續動作空間的對齊。

## §1 · 架構 / Core Mechanism
| 改動維度 | 前作常見設計 | 本法設計 | 工程意圖 |
|---|---|---|---|
| 狀態空間 | 原始價格/技術指標堆疊 | 波動率標準化多週期回報（1天、1個月、2個月、3個月、6個月和1年） | 消除異質波動率對 Q-network 的梯度干擾 |
| 訓練數據 | 僅依賴歷史實盤 | 真實數據 + GBM/VG 模擬數據擴充 | 緩解金融時間序列樣本稀缺導致的探索崩潰 |
| 獎勵函數 | 複雜 Sharpe/Sortino 代理 | 當前動作與價格百分比變動的乘積 | 保持獎勵稀疏性與 PnL 直接對齊，降低 reward hacking |

⚡ **Eureka 一句話 trick**：將多週期風險調整回報構建為狀態空間，結合 GBM/VG 模擬數據擴充訓練集，以離散持倉動作與價格變動乘積定義獎勵函數。
**直覺**：狀態空間本質是「波動率歸一化的動能截面」，代理學習的不是絕對價格方向，而是風險調整後的相對優勢。

```
[市場數據] → (EWMA Vol Scaling) → [狀態 S_t] → DQN Q-network → [動作 A_t ∈ {-1, 0, 1}]
                                                      ↓
                                              [環境反饋] → 價格 P_{t+1} → 獎勵 R_t = A_t * ΔP%
                                                      ↓
                                              [經驗回放] → 梯度更新
```

## §2 · 數學層
📌 **Napkin Formula**：
$R_t = a_t \cdot \frac{P_{t+1}-P_t}{P_t}$
$S_t = \left[ \frac{r_{1d}}{\sigma_{1d}}, \frac{r_{1m}}{\sigma_{1m}}, \dots, \frac{r_{1y}}{\sigma_{1y}} \right]$ （其中 $\sigma$ 為 63天 指數加權移動標準差）

**直覺**：獎勵函數將離散持倉與即時價格變動線性綁定，避免非凸獎勵地形；狀態縮放確保不同波動率 regime 下的特徵量綱一致，使 DQN 的 TD-error 收斂更穩定。訓練細節（如 ε-greedy 衰減、目標網絡更新頻率）導讀未披露。

## §3 · 數據層
- **資料規模/頻率/市場/時段**：E-mini S&P 500 連續、滾動調整期貨合約；日頻（隱含於 1天 回報與 63天 窗口）；跨資產擴展至商品、股票指數、固定收益和匯率。
- **怎麼來**：真實價格序列 + 從真實數據估計參數後生成的 GBM/VG 模擬數據。
- **樣本外與容量假設**：樣本外劃分策略未披露；容量假設受限於單一指數期貨流動性，未驗證多資產並行時的資金分配瓶頸。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations**：導讀截斷前僅提及金融市場應用 RL 面臨「數據特性、探索與利用的平衡以及缺乏基準」等挑戰，具體限制未展開。
**6.2 推斷的隱含假設**：
- **Regime 依賴**：GBM/VG 模擬數據無法捕捉極端尾部事件與流動性枯竭，策略在波動率跳躍時可能失效。
- **成本模型**：雖提及交易成本由比例成本和固定成本組成，但獎勵函數未內建成本懲罰，代理可能過度交易。
- **數據泄漏**：模擬數據參數從真實數據估計，若樣本外劃分不當，易產生前瞻偏差。
- **容量/流動性**：離散動作 {-1, 0, 1} 假設市場可無摩擦承接全倉，未驗證大資金下的衝擊成本。

## §8 · For the Reader
- **因子研究員**：關注狀態空間的波動率標準化邏輯，可將其轉化為傳統動能因子的風險調整版本，替代原始價格序列輸入。
- **組合配置/風控**：離散動作 {-1, 0, 1} 便於與現有的倉位限制與風險預算模塊對接，但需補齊交易成本內生化以避免過度周轉。
- **RL 策略開發**：GBM/VG 模擬擴充是有效的冷啟動手段，但建議在獎勵函數中引入風險懲罰項（如波動率目標或最大回撤約束），並過渡至連續動作空間以匹配真實執行。
