**TL;DR:** ①將Actor-Critic架構引入加密貨幣配對交易，以連續值代理倉位比例與開平倉時機。②核心trick為多部分獎勵塑形（組合收益+區域行動獎勵+調倉懲罰）結合動態閾值調整，使Agent自適應高波動市場。③此設計直接優化「组合执行优化」軸，將傳統離散閾值觸發升級為連續風險敞口管理。④導讀未給量化結果。

**X-Ray.** 本方法在五軸Pareto前沿上，以「全自动黑盒」換取「高频日内」的執行彈性。它解了傳統配對交易的兩個工程坑：一是閾值靜態導致的滯後與假突破（改用動態閾值與Zone劃分）；二是離散倉位控制無法平滑對沖波動（改用連續動作空間與調倉懲罰）。然而，其envelope受限於單對資產的協整假設與短樣本外，在跨市場流動性枯竭或結構性斷裂時，RL策略的連續倉位調整可能放大滑點與資金費率成本。對量化讀者而言，此框架的價值不在於直接實盤，而在於提供了一套可插拔的Reward Shaping模板，可對接至多腿組合或期貨基差策略，但需嚴格對沖交易成本與數據前視偏差。

## §1 · 架構 / Core Mechanism
**1.1 三大改動 vs 前作**
| 改動維度 | 前作 (Gatev et al., 2006 / 傳統規則) | 本方法 |
|---|---|---|
| 閾值機制 | 靜態開平倉閾值 | 動態閾值調整（適應市場波動） |
| 倉位控制 | 離散全倉/空倉 | 連續動作空間（-1至1代表組合百分比） |
| 獎勵信號 | 單一盈虧反饋 | 多部分獎勵塑形（組合收益+區域行動獎勵+調倉懲罰） |

**1.2 ⚡ Eureka**
將「何時交易」與「交易多少」耦合為單一連續動作輸出，並用Zone劃分與調倉懲罰約束探索空間，使RL代理在無預設規則下自發學習均值回歸節奏。

**1.3 信息流 ASCII**
```
[Market Data] -> [Sliding Window] -> [Z-Score Spread & Zone]
                          |
                          v
[Observation Space] -> [Actor Network] -> [Action (-1 to 1)] -> [Position & Cash]
                          ^                      |
                          |                      v
[Critic Network] <- [Multi-Part Reward] <- [Environment Feedback]
```

## §2 · 數學層
📌 **Napkin Formula**
$R_t = R_{portfolio} + \lambda_1 R_{zone} - \lambda_2 |a_t - a_{t-1}|$
$a_t \in [-1, 1]$ (continuous position ratio)
複雜度：Actor-Critic 前向/反向傳播 $O(N_{params})$，訓練依賴PPO/A2C等On/Off-policy算法。

**直覺**：獎勵函數將最終盈虧拆解為「方向正確性（Zone）」與「執行平滑度（調倉懲罰）」，避免代理陷入頻繁刷單或極端倉位。連續動作空間直接輸出目標持倉比例，跳過傳統離散網格搜索。
**Loss/訓練細節**：採用Actor-Critic架構，具體算法對比PPO/A2C/DQN/SAC；訓練期為2023年10月至11月，測試期為12月；使用網格搜索優化窗口大小與初始閾值。

## §3 · 數據層
- **資料規模/頻率/市場/時段**：Binance交易所 BTC-GBP 與 BTC-EUR 交易對；考慮1分鐘、3分鐘、5分鐘交易間隔；形成期為2023年10月至11月，測試期為12月。
- **怎麼來**：歷史價格數據經Engle-Granger協整檢驗與皮爾遜相關係數篩選；價差經滑動窗口回歸並標準化為均值为0，标准差为1的z-score。
- **樣本外與容量假設**：僅1個月樣本外測試；未披露滑點模型與真實訂單簿深度；假設無限流動性與固定交易成本（0.02%、0.01%、0%）。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations**
僅討論未來擴展至多腿策略與優化配對形成，未明確列出當前局限性。

**6.2 推斷的隱含假設**
- **Regime 依賴**：協整關係在加密貨幣高波動下極易斷裂，滑動窗口可能滯後於結構性變化。
- **容量/成本**：假設固定比例交易成本，未建模訂單簿深度與滑點；連續倉位調整在流動性枯竭時會產生巨大衝擊成本。
- **數據泄漏/前視**：網格搜索與動態閾值若未嚴格隔離訓練/測試集，易引入前視偏差；z-score標準化依賴歷史窗口均值/標準差，存在潛在信息泄漏。
- **Survivorship**：僅測試BTC-GBP/EUR，未覆蓋低流動性山寨幣對，存在倖存者偏差。

- **正確答**：「本方法的核心不在於預測價差方向，而在於通過連續動作空間與調倉懲罰優化執行路徑；實盤需嚴格對沖交易成本與協整斷裂風險，獎勵塑形中的Zone劃分是防止過擬合的關鍵。」

**7.1 可證偽預測**
若2024年Q3加密貨幣市場出現持續性單邊趨勢（非均值回歸），該策略的年化收益與夏普比率將顯著劣於靜態閾值基線，且調倉懲罰無法抵消協整破裂帶來的持續虧損。（預測日期：2024-09-30）

## §8 · For the Reader
- **因子研究員**：將Zone劃分與動態閾值抽象為可回測的離散因子，用於傳統統計套利框架的觸發條件優化。
- **高頻執行**：關注連續動作空間的實際下單邏輯；需將RL輸出的目標倉位比例拆解為TWAP/VWAP執行算法，並加入滑點懲罰項。
- **RL 策略研究員**：直接複用多部分獎勵塑形（Portfolio + Action + Penalty）模板，替換環境為期貨基差或跨市場套利，驗證Reward Shaping的泛化性。
- **研究學生**：重點學習Actor-Critic在金融連續動作空間的實現細節，以及Engle-Granger協整檢驗與滑動窗口在數據預處理中的正確隔離方式。
