**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 將對沖基金架構映射為 LLM 多智能體系統。② 核心 trick 是預算分配(BAC)、經驗分享(ESC)、極端市場(EMC) 三類會議機制與記憶迭代。③ 對「多智能體博弈」軸★，用組織協作替代單兵算法的崩潰模式。④ 導讀給出 ARR 71.60% / SR 2.41 / MDD 14.21%。

**X-Ray.** 本框架將「對沖」從數學優化降維至組織行為學，直接切中 LLM/RL 單智能體在極端波動下的崩潰模式。它解了提示詞漂移與風險預算硬編碼的工程坑，但打不開高頻執行與低延遲決策的 envelope。對量化讀者而言，其價值不在預測精度，而在將風險管理模組化、可審計化；後續應將此架構與傳統因子組合或 RL 執行層解耦，而非全盤託管。

## §1 · 架構 / Core Mechanism
| 改動維度 | 前作 (單智能體/RL/LLM) | HedgeAgents 改動 |
|---|---|---|
| 決策主體 | 單一模型直出倉位 | 三領域專家(Dave/Bob/Emily) + 經理(Otto) 分工 |
| 資源分配 | 靜態權重或 RL 策略網絡 | 動態預算分配會議(BAC)，30天週期重算 |
| 經驗累積 | 無狀態或單次 Replay Buffer | 投資反思記憶(M_IR) → 通用經驗記憶(M_GE) 迭代 |

⚡ **Eureka:** 對沖不是單兵算法，是組織行為學。
**信息流 ASCII:**
```
[市場數據/新聞] → [專家分析] → [BAC會議] → [Otto權重優化] → [執行]
       ↑                                      ↓
[ESC會議] ← [M_IR反思] ← [M_GE歸檔] ← [週期結束]
       ↑
[EMC會議] ← [波動>5%或累計>10%] ← [Otto+危機智能體] → [動態降倉/對沖]
```

## §2 · 數學層
📌 **Napkin Formula:**
$$\max_{w} \left( I_{etr} - \lambda_1 I_{pr} - \lambda_2 I_{cvar} \right) \quad \text{s.t.} \quad \sum w_i = 1, \ w_i \ge 0$$
**直覺:** BAC 會議將多資產配置轉為帶風險厭惡係數的凸優化問題。$I_{etr}$ 抓收益，$I_{pr}$ 控協方差，$I_{cvar}$ 鎖尾部風險。
**複雜度/訓練:** 無梯度回傳。複雜度來自 LLM 推理（每次會議）+ 二次規劃求解權重。Loss 由提示詞模板與歷史回報隱式定義；訓練依賴 M_GE 的 LLM 摘要更新，屬離線記憶強化。

## §3 · 數據層
- **規模/頻率/市場/時段:** 比特幣、外匯、道瓊斯成分股。日頻。2015 年 1 月 1 日至 2023 年 12 月 31 日。
- **來源:** 雅虎財經 + Alpaca 新聞 API。
- **特徵:** 每日開高低收、成交量、調整後收盤價、每日新聞更新、60 個標準技術分析指標。
- **樣本外與容量假設:** 導讀未明確劃分 OOS 集，但區間覆蓋 2022 年動盪期；日頻調倉理論容量高，但受 LLM API 成本與會議延遲約束，實盤需壓縮會議開銷。

## §5 · 評測 / Benchmark
| 數據集/市場 | Metric | 前SOTA (導讀逐字基線) | 本方法 | Δ |
|---|---|---|---|---|
| BTC/FX/DJIA | ARR | FinAgent | 71.60% | 高出 33.75% |
| BTC/FX/DJIA | TR | FinAgent | 405.34% | 高出 54.72% |
| BTC/FX/DJIA | SR | FinGPT | 2.41 | 高出 24.49% |
| BTC/FX/DJIA | CR | FinGPT | 11.02 | 高出 44.28% |
| BTC/FX/DJIA | MDD | FinGPT | 14.21% | 低 16.76% |
| BTC/FX/DJIA | ENT | FinAgent | 3.13 | 高出 6.85% |
| BTC/FX/DJIA | ENB | FinAgent | 1.53 | 高出 8.51% |

**解讀:** Δ 在 ARR/TR 主要反映 BAC 的槓桿與倉位放大效應，非純 Alpha 提升；MDD 下降與 ENT/ENB 上升屬真實風險分散能力。需警惕日頻新聞與調倉間的前瞻偏差，且 LLM API 成本與滑點未計入淨值。消融實驗中，移除 ESC 導致 ENT 下降 16.63%，移除 BAC 導致 ARR 下降 37.75%，移除 EMC 導致 MDD 增加 71.93%，佐證三模組為獨立能力而非冗余。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations:** 依賴 LLM 質量與成本；提示詞敏感；BAC 30天週期對閃崩反應滯後；開源模型在部分指標上弱於閉源。
**6.2 推斷的隱含假設:** 
- Regime 依賴: 假設新聞情緒與日頻價格聯動結構穩定，EMC 觸發閾值(5%/10%) 未隨波動率 regime 動態調整。
- 成本/容量: 未計入交易成本、滑點與 LLM 推理延遲；實盤容量受會議開銷限制。
- 數據泄漏: 每日新聞與日頻收盤價若未嚴格按時間戳對齊，易產生前視偏差。
- Survivorship: DJIA 成分股歷史回溯未說明是否處理退市/調樣本偏差。

🎤 **Interview Tip:** 
- ✅ 正確答: 「多智能體不提升預測精度，而是將風險預算、經驗記憶與極端應對解耦為可審計模組，降低單點策略的尾部崩潰概率。」
- ❌ 錯答: 「多智能體能互相投票，所以預測更準，能替代傳統因子模型。」

**7.1 可證偽預測:** 至 2025-12-31，若 LLM API 延遲未降至 <200ms/次且成本未壓縮，該架構在實盤日頻調倉中的淨值曲線將因會議開銷與滑點顯著劣於靜態風險平價組合。

## §8 · For the Reader
- **因子研究員:** 將 M_GE 視為「可編程因子庫」，用 ESC 機制自動化因子衰減檢測與輪動記錄，替代手動日誌。
- **LLM-Agent 開發者:** 別堆疊 Agent 數量；優先實現 BAC 的權重優化閉環與 EMC 的觸發降倉邏輯，這是實盤存活的錨點。
- **組合配置/風控:** 將此架構作為「策略層風險閥門」，而非信號源；用 $I_{cvar}$ 與協方差矩陣做組合級限倉，與底層 Alpha 解耦。
