# FinVision 解構（FinVision）

> **發布**：2024-11-15 · （無 venue）
> **QuantML 導讀**：[FinVision: 基于多模态多智能体框架的股票市场预测系统](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247487733&idx=1&sn=e95f5aa14a6e742614f1f3aeba626605&chksm=ce7e77ebf909fefd8a81e646e374c3b12b5a2093bc3b72acbf27353f0c2f502bc885aeef0045#rd)
> **核心定位**：落點於「生成式大模型 × 多智能体博弈」軸，解決傳統 CV/RL 將金融數據過度簡化為單變量或黑箱決策的解釋性與多模態整合 gap。

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** 提出 FinVision 多模态多智能体框架，結合 LLM 視覺與文本推理，實現可解釋的日频股票交易決策。核心 trick 是引入摘要、技術分析與反思三模塊協同的 Agent 工作流，將 K 線圖與新聞文本統一為多模態輸入，並內嵌風險管理。這對「多智能体博弈」軸的關鍵在於用 LangGraph 狀態機替代端到端 RL 訓練，以推理換訓練成本。實證顯示在 AAPL 上年化收益率達 14.79%，夏普比率 1.20，優於同期買入持有策略。

**X-Ray.** FinVision 將 Alpha 生成從「參數優化」轉向「提示詞工程與狀態路由」，本質是 LangGraph 驅動的條件生成系統。它解了傳統 CV 看線忽略文本語境、RL 黑箱難調參的工程坑，但代價是將市場風險轉移至 LLM 的幻覺與 API 延遲。該框架打不開高頻或低流動性股票的 envelope，因其依賴日频新聞與圖表特徵的滯後性。對量化讀者而言，其價值不在直接實盤，而在提供一套可插拔的多模態特徵提取器，可與傳統因子模型或執行算法解耦組合，作為信號前置過濾器。

## §1 · 架構 / Core Mechanism
| 改動維度 | 前作/傳統做法 | FinVision 改動 | 量化意義 |
|---|---|---|---|
| 數據輸入 | 單一矩陣或單變量情緒得分 | K 線圖 + 新聞文本多模態統一 | 保留圖表空間特徵與文本語境，避免信息降維損失 |
| 決策邏輯 | 端到端 RL 梯度下降或規則系統 | LangGraph 狀態機路由 + 多智能體對話 | 模塊解耦，可單獨替換或升級單一節點，降低全鏈路重訓成本 |
| 風險控制 | 外置止損或 RL 獎勵函數硬編碼 | 內嵌於最終決策模塊的頭寸規模預測 | 將風險管理從後驗監控轉為先驗約束，符合實盤風控前置需求 |

⚡ **Eureka:** 用 LangGraph 狀態機將「摘要-技術分析-反思-決策」串聯，替代端到端 RL 訓練，以提示詞路由實現模塊解耦。

```
[新聞文本] → (摘要模塊) → [精煉事實]
[K線/指標圖] → (技術分析模塊) → [圖表模式/趨勢]
[歷史交易/信號圖] → (反思模塊) → [績效洞察/信號反饋]
       ↓
[最終決策模塊] → 預測動作 + 頭寸比例 + 解釋 → [執行/狀態更新]
```

## §2 · 數學層
📌 **Napkin Formula:** 無傳統可微 Loss。核心為狀態轉移 $S_{t+1} = \mathcal{F}(S_t, \text{Prompt}_{\text{sum}}, \text{Prompt}_{\text{tech}}, \text{Prompt}_{\text{refl}}; \theta_{\text{LLM}})$，計算複雜度 $O(N_{\text{agents}} \times T_{\text{tokens}})$。
**直覺:** 將交易決策建模為條件語言生成任務，透過溫度參數控制探索與利用的權衡，以文本輸出直接映射為交易指令與倉位權重。
**Loss/訓練細節:** 導讀未披露微調或 RLHF 過程，僅說明使用 GPT-4o-mini (temp 0.3) 與 o1-mini (temp 1) 進行推理與決策，依賴提示詞工程與少樣本引導而非參數更新。

## §3 · 數據層
- **市場/標的:** 美股三大科技股 (AAPL, AMZN, MSFT)
- **頻率/時段:** 日频。測試期為 2023-06-01 至 2023-12-29（訓練期 2023-04-01 至 2023-05-31）
- **數據來源:** 雅虎財經新聞、每日 K 線圖、技術指標（SMA10/50, RSI14, BB20, MACD, Volume）
- **樣本外與容量假設:** 導讀未明確說明跨市場/跨週期驗證，僅覆蓋單一科技股牛市階段；LLM 推理延遲與 API 限流假設不適合大資金或高頻場景。

## §5 · 評測 / Benchmark
| 數據集/市場 | Metric | 前SOTA | 本方法 | Δ |
|---|---|---|---|---|
| AAPL | ARR | 買入持有 13.56% | 14.79% | 1.23pp |
| AAPL | SR | 買入持有 0.67 | 1.20 | 0.53 |
| MSFT | ARR | 買入持有 22.27% | 25.57% | 3.30pp |
| MSFT | SR | 買入持有 1.01 | 1.41 | 0.40 |
| AMZN | ARR | 買入持有 43.57% | 42.14% | -1.43pp |
| AMZN | SR | 買入持有 1.37 | 1.72 | 0.35 |
| AMZN | MDD | 買入持有 17.45% | 12.09% | -5.36pp |
| RL (PPO/DQN) | SR | 未披露 (導讀僅述為負值) | 1.20~1.72 | N/A |
| FinAgent | ARR/SR | 未披露 (導讀僅述優於本法) | 14.79%~42.14% | N/A |

**解讀:** Δ 主要來自風險控制（MDD 縮減）與牛市趨勢跟隨，非絕對 Alpha 爆發。RL 模型負 SR 反映傳統 RL 在日频小樣本易過擬合與獎勵稀疏問題。FinAgent 雖絕對收益更高，但訓練週期長達一年，本法以推理成本換取開發效率。需注意測試期僅 7 個月且全為科技股，Δ 可能含強勁行業 beta 與 regime 貢獻，非純淨 alpha。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations:** 訓練時間雖縮短至兩個月，但性能仍不及 FinAgent；依賴 LLM API 成本；未涵蓋高頻或低流動性資產；反思模塊需歷史數據積累，冷啟動期表現未說明。
**6.2 推斷的隱含假設:** 
- **Regime 依賴:** 測試期為 2023 年科技股單邊牛市，未驗證震盪/熊市或風格切換下的魯棒性。
- **數據泄漏風險:** 新聞摘要與 K 線生成若未嚴格對齊時間戳（如盤後新聞用於次日開盤決策），易引入前瞻偏差。
- **模型幻覺:** 假設 LLM 視覺模塊能穩定識別技術圖表模式，未驗證模型對雜訊圖表的過度擬合或錯誤信號生成。

🎤 **Interview Tip:** 
- **正確答:** 「本法本質是 LangGraph 驅動的條件生成系統，用多智能體對話替代梯度下降，優勢在於可解釋與模塊解耦，劣勢在於推理成本與 regime 敏感。實戰應將其降維為特徵提取器，而非直接下單引擎。」
- **錯答:** 「LLM 能直接預測股價，取代傳統因子模型。」

**7.1 可證偽預測:** 若 2025-Q3 美股進入高波動震盪市，本法夏普比率將回落至 0.8 以下，且反思模塊因歷史信號失效產生過度交易。

## §8 · For the Reader
- **因子研究員:** 提取其「摘要模塊」的文本特徵與「技術分析模塊」的圖表模式，作為非線性因子輸入傳統組合優化，避免直接依賴 LLM 下單。
- **高頻執行:** 本法不適用。但其「反思模塊」的績效歸因邏輯可移植至執行算法的滑點監控與訂單拆分策略，用於盤後歸因與參數動態調整。
- **LLM-Agent 開發者:** 參考 LangGraph 狀態機設計，將交易信號生成與風險檢查解耦，避免單一大模型承擔全鏈路決策，提升系統可維護性與故障隔離能力。
