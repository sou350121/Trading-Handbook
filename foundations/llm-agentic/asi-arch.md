# ASI-ARCH 解構

> **發布**：2025-08-04 · （無 venue）
> **QuantML 導讀**：[AI自我进化：模型架构探索的AlphaGo时刻](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247491248&idx=1&sn=d97b2d09033ad0fc38b76018dd01a114&chksm=ce7e79aef909f0b897a2e7f2c165dddf5de7c89ff597c3cf45e99d97269cbfac7290d34ccae9#rd)
> **核心定位**：落點於「元学习搜索 × Agent自主演进」軸，將傳統 NAS 的預定義空間限制，升級為 LLM 驅動的多智能體閉環科研。解了量化/ML 架構設計中人類先驗偏見與迭代帶寬的 prior gap，驗證了「科學發現可計算擴展」的假設。

**五軸座標**

| 數據模態 | 時間尺度 | 學習範式 | Alpha機制 | 人機協作 |
|:-:|:-:|:-:|:-:|:-:|
| `文本另类` | `跨周期` | `元学习搜索` | `端到端表征` | `Agent自主演进` |

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 構建全自主多智能體系統 ASI-ARCH，透過 LLM 驅動進化搜索自動發現線性注意力架構。② 核心 trick 在於複合適應度函數（量化性能 Sigmoid 變換 + LLM 評委定性打分）與「認知庫檢索+情境對比」的雙軌知識注入。③ 對「元学习搜索」軸具里程碑意義，證明架構創新可脫離人類先驗空間，實現從模仿到抽象推理的自主躍遷。④ 導讀未給量化結果（僅給 GPU 小時與架構數量，無具體性能指標對比）。

**X-Ray.** 放回五軸 Pareto，ASI-ARCH 將「人機協作度」推向 Agent 自主演進的極端，代價是計算開銷與評審主觀性。它解了傳統 NAS 的兩大工程坑：一是單一 loss 導致的 reward hacking，二是人類設計空間的局部最優。系統透過 Sigmoid 壓縮性能差異並引入 LLM 評委，強制搜索路徑偏離「暴力堆參數」，轉向組件級微創新。然而，其打不開的 envelope 在於：LLM 評委的打分一致性與領域知識邊界（文本/代碼訓練數據的 bias 會直接映射為架構偏好）；此外，「認知庫」依賴人類文獻，本質仍是「站在巨人肩膀上的自動化」，而非從零物理定律推導。對量化讀者而言，此框架的啟示不在於直接套用線性注意力，而在於「複合適應度設計」與「情境對比分析」可移植至因子挖掘/策略搜索：當單一回測指標失效時，引入結構複雜度懲罰與跨 regime 對比，能有效過濾過擬合。但需警惕 LLM-as-judge 在金融數據上的幻覺與前瞻偏差風險。

## §1 · 架構 / Core Mechanism
**1.1 三大改動 vs 前作**
| 維度 | 傳統 NAS / 單智能體搜索 | ASI-ARCH 改動 |
|---|---|---|
| 適應度引導 | 單一量化指標（Loss/Acc） | 複合函數：量化性能 Sigmoid 變換 + LLM 評委定性打分 |
| 知識注入 | 隨機初始化或固定先驗 | 認知庫檢索（人類文獻結構化） + 情境對比分析（父本/兄弟節點消融） |
| 驗證閉環 | 丟棄失敗代碼/手動調參 | 工程師智能體自動捕獲日誌、自我修正代碼，兩階段探索-驗證策略 |

**1.2 ⚡ Eureka Trick**
用 Sigmoid 壓縮性能差異值以放大微小改進並抑制極端值，配合 LLM 評委的結構性打分，強制搜索脫離「獎勵黑客」與參數堆疊陷阱。

## §2 · 數學層
📌 **Napkin Formula:**
`Fitness = σ(ΔPerf) + α·Score_LLM` （σ為 Sigmoid 變換，ΔPerf 為相對基線性能差異，Score_LLM 經歸一化；原文公式具體形式截斷/未顯示）
**直覺:** Sigmoid 壓縮極端值防止單一指標主導，LLM 項注入結構/複雜度/創新性先驗，形成多目標 Pareto 搜索。
**Loss/訓練細節:** 探索階段限 20M 參數、1B tokens；驗證階段放大至 340M/400M 參數，頂尖模型最終在 15B tokens 完整訓練。未披露具體 Loss 函數形式與優化器超參。

**解讀:** Δ 的「真 capability」在於系統能在不增加參數規模的前提下，透過組件級創新（如門控路由、並行 Sigmoid 替代 Softmax）突破基線。但導讀未給出統計顯著性檢驗、計算開銷對比或推理延遲數據，部分「超越」可能源自測試集分佈偏差或 LLM 評委的過擬合偏好，需警惕未計入工程優化成本（如未寫 Triton 核心）的虛高表現。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations:** ① 僅從單一基線（DeltaNet）初始化，未探索全新架構家族；② 受資源限制，未對框架組件（認知 vs 分析）進行細粒度消融；③ 未為新架構編寫定制加速內核，缺乏計算效率與延遲基準測試。
**6.2 推斷的隱含假設:** ① **Regime 依賴:** LLM 評委的打分邏輯高度依賴訓練數據分佈，若遷移至非文本/非代碼領域（如高頻量價數據），評審標準可能失效；② **容量/成本:** 20,000 GPU 小時的閉環搜索成本極高，不適合低延遲實時迭代場景；③ **數據泄漏/前瞻:** 認知庫提取自人類文獻，若文獻本身包含未公開的數據處理技巧，系統可能隱式繼承前瞻偏差；④ **代碼健全性:** 複雜度檢查僅限於 TBD 閾值，未覆蓋記憶體洩漏或並行計算死鎖等工程失效模式。

## §8 · For the Reader
- **因子研究員:** 借鑒「情境對比分析」邏輯，在因子挖掘中引入父因子/兄弟因子的消融對比，而非僅看單因子 IC/IR，可過濾共線性與過擬合。
- **高頻執行/系統架構:** 關注「兩階段探索-驗證」策略，實盤前可先用低頻/小樣本快速篩選策略池，再放大至全量數據與真實滑點模型驗證，降低算力浪費。
- **RL 策略/Agent 開發:** 複合適應度函數設計可直接映射至 Reward Shaping：將純收益指標替換為「收益 Sigmoid 壓縮 + 風險/複雜度懲罰 + 人工/LLM 結構評分」，避免策略收斂至極端杠桿或過交易。
- **研究學生:** 警惕「LLM-as-judge」的幻覺與基準漂移，在復現時需固定評委模型版本與 Prompt 溫度，並記錄打分分佈的長期穩定性。
