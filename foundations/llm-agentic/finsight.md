# FinSight 解構（FinSight）

> **發布**：2025-10-26 · （無 venue）
> **QuantML 導讀**：[人大高瓴 x 智源 ｜ FinSight：会思考、懂绘图、善写作的专业AI金融分析师](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247492124&idx=1&sn=869727f5b39cf4fb2dcca8a421c00c4a&chksm=ce7d8502f90a0c1404fbd70c29fdde699136695eefa6c1c6b77c57e2f7d631f0a1132576c8f7#rd)
> **核心定位**：落點於「多模態 × 中長週期 × 生成式大模型 × 多智能體博弈 × Agent自主演進」軸心，解決了傳統 LLM+Search 單次生成導致的分析鏈斷裂與多模態幻覺問題，將金融研報生產從「靜態文本拼裝」推向「代碼驅動的動態變數空間」。

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 提出 FinSight 多智能體框架，自動化生成高質量多模態金融研報。② 核心 trick 為 CAVM 架構統一數據/工具/智能體至可編程變數空間，結合迭代視覺增強與兩階段寫作。③ 對「多智能體博弈」軸★，以代碼執行替代固定 Prompt 鏈，實現動態搜索與專業圖表生成。④ 導讀給出總分 8.09，顯著領先基線。

**X-Ray.** FinSight 將研報生成的 Pareto 前沿從「單次文本生成」推向「代碼驅動的變數空間迭代」。它解了兩個舊工程坑：一是靜態 Pipeline 無法根據中間發現調整策略，二是 VLM 缺乏閉環反饋導致圖表專業度不足。其 envelope 明確受限於高延遲與高 Token 成本，不適用於日頻 Alpha 生成或高頻執行，而是定位於季頻深度覆盤與盡調自動化。對量化讀者而言，CAVM 的變數映射邏輯與兩階段解耦（分析鏈→報告）可直接移植至因子敘事生成與非結構化數據結構化提取，但需警惕 LLM-as-Judge 評估帶來的基準漂移風險。

## §1 · 架構 / Core Mechanism
**1.1 三大改動 vs 前作**
| 維度 | 前作 (LLM+Search / 單輪生成) | FinSight |
|---|---|---|
| 數據/工具整合 | 固定 Prompt 鏈，靜態調用 | CAVM 架構：統一映射至可編程變數空間，代碼驅動動態執行 |
| 多模態輸出 | 純文本，缺乏圖表或一次性生成 | 迭代式視覺增強：VLM 批判反饋 → 代碼修正 → 專業級複合圖表 |
| 寫作流程 | 單次生成，邏輯易斷裂 | 兩階段寫作：先濃縮「分析鏈」片段，再擴展為上下文感知報告 |

**1.2 ⚡ Eureka 一句話 trick + 直覺**
CAVM 將所有數據、工具與智能體映射為可編程變數，使 LLM 的代碼生成能力直接驅動工作流編排；直覺上是用「執行環境」替代「提示詞工程」，讓模型在變數空間中動態搜索與迭代。

**1.3 信息流 ASCII 圖**
```
[異構數據/新聞/財報] → Data Collection → 結構化多模態記憶庫
       ↓
[CAVM 變數空間] ↔ 動態搜索/工具調用 ↔ 多輪交互分析
       ↓ (輸出: 分析鏈片段)
[Two-Stage Writing] → 片段擴展 → 圖文引用綁定 → 專業研報
       ↑
[Iterative Vision-Enhanced] ← VLM 反饋 ← 代碼生成圖表
```

## §5 · 評測 / Benchmark
| 數據集/市場 | Metric | 基線A (Gemini-2.5-Pro DR) | 基線B (OpenAI DR) | 本方法 (FinSight) | Δ |

| 同上 | 結論一致性 | 7.10 | 未披露 | 6.85 | -0.25 / TBD |
| 同上 | 可視化 (圖表表現力) | 未披露 | 未披露 | 9.00 | 未披露 |

**解讀:** 
- Δ 在總分與可視化上的提升屬真 Capability：迭代視覺增強與 CAVM 代碼驅動直接彌補了自動化圖表與專家級視覺的差距。
- 結論一致性 Δ 為負屬設計取捨：FinSight 優先全面數據搜集以追求更深刻洞察，得出更多數據驅動發現，而非傳統搜索生成的簡化結論，非過擬合。
- 樣本僅 20 個，統計效力有限；導讀未計入 API 成本與延遲，實際部署的 Breakeven 成本未披露。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations**
- 缺乏標準金融報告評估基準，依賴自建 20 樣本與 LLM-as-Judge。
- 生成過程 Token 與搜索查詢量極高（平均每份報告執行983.2次搜索查詢和469.8次頁面瀏覽），成本與延遲未優化。
- 專注於報告生成，不直接輸出交易信號或價格預測。

**6.2 推斷的隱含假設**
- **Regime 依賴:** 假設基本面邏輯與圖表解讀在跨市場/跨時段具有穩定性，未處理結構性斷點。
- **容量/成本:** 隱含假設研究機構可承擔高 Token 成本與小時級延遲，不適用於日頻或高頻場景。
- **數據泄漏風險:** 動態搜索若未嚴格對齊時間戳，可能引入未來信息（Look-ahead bias）。
- **評估偏差:** 依賴 Gemini-2.5-Pro 評分，存在 Judge 模型偏好與基準漂移風險。

## §8 · For the Reader
- **因子研究員:** 將 CAVM 的變數映射邏輯移植至因子敘事生成，用「分析鏈」兩階段解耦替代手動撰寫，提升非結構化數據到結構化因子的轉換效率。
- **LLM-Agent 開發者:** 參考 Iterative Vision-Enhanced 的 VLM 閉環反饋模式，解決多模態輸出中的幻覺與格式崩潰問題，建立可編程的 Agent 狀態機。
- **組合配置/研究學生:** 理解「先分析、後寫作」如何避免單次生成的邏輯斷裂；在實盤中將此框架用於基本面盡調自動化，而非直接接入交易執行層。
