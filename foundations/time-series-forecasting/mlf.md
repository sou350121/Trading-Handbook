# MLF 解構（MLF）

> **發布**：2025-08-06 · KDD 2025
> **QuantML 導讀**：[KDD 25 | 复旦&蚂蚁：多周期时间序列预测框架](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247491262&idx=1&sn=d941acef9fc21a7161622133f53d2081&chksm=ce7e79a0f909f0b630fb89d694feaf1ac0a4f09a9c99bf334db03b3b9ce75a11e10886b1d3da#rd)
> **核心定位**：落點於「跨周期監督回歸」與「端到端黑盒」軸；解決了金融時序中單一視窗無法兼顧短波長動盪與長週期趨勢的結構性割裂，同時規避了多週期粗暴拼接導致的注意力冗餘與計算膨脹。

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 將不同長度歷史視窗並行輸入，透過自適應分塊與冗餘過濾實現多週期語義解耦。② 核心 trick 在於 MAP 均衡塊數、IRF 剔除跨週期重疊信息、LWI 動態加權，並用 Patch Squeeze 降維提效。③ 對「跨周期」軸★：將週期選擇成本轉化為可學習的集成權重，避免人工滑窗調參。④ 線上A/B測試WMAPE相對提升5.16%，推理速度較Scaleformer快3.6倍。

**X-Ray.** MLF 將多週期輸入從「特徵工程」升級為「架構內生機制」。傳統 TSF 依賴單一眼窗，本質是將週期選擇的組合優化問題外包給研究員；MLF 透過 MAP 與 IRF 將不同頻率信號在嵌入空間正交化，再用 LWI 完成動態路由。這在「端到端表徵」軸上切開了 PatchTST 等單週期模型的效能天花板，同時以 Patch Squeeze 的自監督壓縮對沖了多視窗帶來的計算膨脹。對量化讀者而言，其價值不在於單純的 MSE 下降，而在於提供了一套可自動適應市場 regime 切換的週期路由機制，大幅降低滑窗長度調參的過擬合風險與訓練成本。

## §1 · 架構 / Core Mechanism
| 維度 | 前作 (PatchTST / FiLM) | MLF 改動 |
|---|---|---|
| 週期處理 | 單一眼窗或簡單線性拼接 | MAP 自適應分塊 + IRF 跨週期冗餘過濾 |
| 結果集成 | 固定平均或簡單加權 | LWI 可學習動態加權集成 |
| 計算效率 | 隨輸入長度線性/二次膨脹 | Patch Squeeze 自監督降維壓縮 |

⚡ **Eureka:** 用「塊數均衡」與「冗餘正交化」取代人工週期選擇，讓模型自動在短波長動盪與長週期趨勢間動態路由。

```
[多週期輸入] → [MAP均衡分塊] → [Patch Squeeze壓縮] → [Transformer塊]
    ↓                              ↓
[IRF冗餘過濾] ← [單週期表示分割] ← [堆疊MLF塊]
    ↓
[LWI動態加權] → [最終預測輸出]
```

## §2 · 數學層
📌 **Napkin Formula:**
$\tilde{Z}_i = Z_i - \sum_{j < i} R_{j \to i}$ （IRF 冗餘剔除）
$\hat{Y} = \sum_{i} w_i \hat{Y}_i, \quad w_i = \sigma(\text{MLP}(q, k_i))$ （LWI 加權集成）
複雜度：自注意力 $O(L^2)$ 經 Patch Squeeze 壓縮後降至 $O((L/c)^2)$，$c$ 為壓縮因子。

**直覺:** IRF 在特徵空間做「減法」，強迫模型關注非重疊週期的獨特信號；LWI 將週期權重轉化為條件概率分佈，隨市場狀態動態切換主導頻率。
**Loss/訓練:** 主任務預測損失 + Patch Squeeze 的重構損失 (reconstruction loss) 聯合優化，端到端反向傳播。

## §3 · 數據層
- **資料規模/頻率/市場:** 螞蟻財富基金銷售數據（日頻）+ 公開數據集 (Electricity, ETT, Weather)。
- **怎麼來:** 每日閉市後自動重訓與推理，預測未來5天銷量。
- **樣本外與容量假設:** 線上A/B測試連續五週；未披露具體樣本量與總交易筆數，假設為高頻重訓的工業級流水線，容量受每日重訓算力與庫存系統延遲約束。

## §5 · 評測 / Benchmark
| 數據集/市場 | Metric | 前SOTA | 本方法 | Δ |
|---|---|---|---|---|
| Fund (螞蟻財富) | 推理速度 | Scaleformer | MLF | 快3.6倍 |
| Fund (螞蟻財富) | 推理速度 | Pathformer | MLF | 快11.8倍 |
| Fund (螞蟻財富) | WMAPE (線上A/B) | PatchTST | MLF | 提升5.16% |
| Fund (螞蟻財富) | GMV (線上A/B) | PatchTST | MLF | 提升0.84% |

**解讀:** Δ 主要來自「效率」與「線上業務指標」，非純學術 MSE/MAE。推理加速 3.6-11.8 倍是真 capability（Patch Squeeze 降維生效），但學術榜單的 MSE/MAE 絕對值導讀未披露，無法判斷過擬合。WMAPE 提升 5.16% 與 GMV 提升 0.84% 屬工業部署實證，已計入線上流量與庫存成本，非回測幻象。需注意 Fund 數據集模式複雜（低趨勢/高峽度），學術公開集（ETT/Weather）的相對增益可能不同。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations:** 未明確列出學術局限性，僅強調部署有效性與消融驗證。
**6.2 推斷的隱含假設:**
- **Regime 依賴:** LWI 權重依賴最長週期輸入的全局特徵向量，若市場出現結構性斷裂（如政策突變），CNN 提取的特徵可能滯後，導致權重分配失準。
- **容量/成本:** 每日重訓與推理依賴穩定算力流水線，若數據頻率提升至分鐘級，Patch Squeeze 的壓縮比可能不足以支撐實時延遲要求。
- **數據泄漏:** MAP 分塊與 IRF 過濾依賴歷史視窗，若未來值與當前視窗邊界存在數據洩漏（如基金申贖確認延遲），WMAPE 優化可能過度拟合短期噪音。
- **樣本分佈:** 導讀指出 33.53%/27.35%/39.12% 的週期優勢分佈接近均分，假設未來樣本維持此分佈；若市場轉為單週期主導，LWI 的動態加權收益將遞減。

## §7 · 對比 & 面試 Tip
| 同軸對手 | 關鍵差異軸 | Open? | Status |
|---|---|---|---|
| PatchTST | 單週期 vs 多週期路由 | 是 | 學術基線 |
| FiLM | 簡單線性拼接 vs IRF正交化+LWI動態加權 | 是 | 學術基線 |
| Pyraformer | 多尺度降採樣 vs 多週期原始視窗並行 | 是 | 學術基線 |

🎤 **Interview Tip:**
- **正確答:** MLF 不是特徵拼接，而是透過 MAP 均衡塊數、IRF 在嵌入空間做冗餘減法、LWI 做條件加權，本質是將週期選擇轉為可學習的路由問題。
- **錯答:** MLF 只是把不同長度的序列拼在一起送進 Transformer，然後加個注意力權重。（忽略 IRF 的正交化與 Patch Squeeze 的壓縮機制）

**7.1 可證偽預測:** 若 2025-Q4 公開數據集（如 ETTm1）上 MLF 的 MSE 未顯著優於 PatchTST（Δ < 1%），則證明其多週期路由機制對低峽度/高趨勢數據泛化有限，僅適用於高波動金融流水線。

## §8 · For the Reader
- **因子研究員:** 將 MAP+IRF 思路移植至多頻率因子合成，替代傳統 IC/IR 加權，降低週期錯配風險。
- **高頻執行:** Patch Squeeze 的自監督壓縮可參考用於盤中特徵降維，但需驗證分鐘級數據的實時推理延遲。
- **組合配置:** LWI 的動態權重機制可直接映射為週期風險預算分配，替代靜態風險平價模型。
- **LLM-agent:** 多週期輸入的「冗餘過濾」思想可用於 Prompt 壓縮，剔除歷史對話中的重複上下文，提升 Token 效率。
