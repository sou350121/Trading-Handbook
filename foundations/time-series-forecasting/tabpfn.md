> **核心定位**：落點於量價表格與跨周期監督回歸，解決了 TabPFN-TS 過度依賴外部時間戳日曆特徵而無法捕獲數據內在複雜/非標準週期的 Prior Gap。

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 將時間序列預測轉化為表格回歸，透過 FFT 提取主導週期並按週期間隔回溯構建特徵矩陣，實現零樣本預測。② 核心 Trick 是徹底摒棄外部日曆特徵，改以數據內在頻譜週期驅動特徵工程。③ 這對「端到端表徵」軸★ 意味著模型不再依賴人工時間編碼，直接從頻域對齊歷史分佈，提升泛化邊界。④ 導讀未給量化結果，僅定性指出多數情境下 MAE 優於 TabPFN-TS。

**X-Ray.** 放回五軸 Pareto，本法將「特徵工程」從人工日曆編碼推向頻域自動對齊，實質是將週期性先驗注入 TabPFN 的 in-context learning 機制。它解了舊工程坑：傳統 TS 基礎模型依賴固定視窗或外部時間戳，面對非標準週期（如學期、產業季調）時特徵對齊失效；本法以 FFT 峰值週期為錨點回溯切片，使表格輸入的列分佈天然匹配數據的內在節奏。預測其打不開的 envelope：對非平穩/週期漂移劇烈的序列，單一 FFT 主導週期會產生頻譜洩漏與特徵錯位；且回溯構建矩陣時若週期非整數倍，截斷將引入資訊損耗。對量化讀者意義：提供了一條「頻域特徵化 + 表格基礎模型」的輕量級零樣本路徑，適合週期穩定、標的眾多的中低頻因子挖掘；但需警惕其未計入交易成本與 regime 切換下的週期失效風險，實盤前必須加入週期穩定性檢驗與動態窗口機制。

## §1 · 架構 / Core Mechanism
**1.1 三大改動 vs 前作**
| 維度 | TabPFN-TS (前作) | 本法 (FFT-TabPFN) | 改動本質 |
|---|---|---|---|
| 特徵來源 | 外部時間戳日曆特徵 (年/月/日 sin-cos) + 運行索引 | 數據內在週期 (FFT 頻譜峰值) + 歷史回溯切片 | 從「外部時間錨」轉向「內在頻域錨」 |
| 週期假設 | 隱含標準週期 (依賴時間戳對齊) | 顯式提取主導週期 (k=1 經驗設定) | 解除對標準日曆的依賴，適配非標準週期 |
| 輸入結構 | 時間戳編碼向量 | 按週期間隔回溯構建的二維特徵矩陣 | 特徵矩陣列分佈直接映射週期節奏 |

**1.2 ⚡ Eureka 一句話 trick + 直覺**
用 FFT 抓出數據「心跳頻率」，按該頻率間隔往回切歷史切片，把時間序列硬轉成 TabPFN 最擅長的表格分佈對齊問題。

**1.3 信息流 ASCII 圖**
```
原始序列 [t0...tN]
      │
      ▼
FFT 頻譜分析 → 選取振幅最高頻率 → 計算主導週期 P
      │
      ▼
按間隔 P 回溯歷史切片 → 構建特徵矩陣 X_train / X_test
      │
      ▼
TabPFN v2.0 (回歸任務, output="mean") → 預測值 ŷ
      │
      ▼
評估: MAE vs 真實值 y_test
```

## §2 · 數學層
📌 **Napkin Formula**:
$P = 1/f_{top}$, 其中 $f_{top} = \arg\max_{f} |\mathcal{F}(x_{hist})|$
特徵構建: $X_{i,j} = x_{t - j \cdot P}$ (按週期 P 回溯索引)
複雜度: FFT $O(N \log N)$ + 特徵矩陣構建 $O(N \cdot k \cdot \text{lookback})$ + TabPFN 推理 $O(N_{samples}^2)$ (in-context 注意力)
直覺: 將時域週期性轉化為表格列的對齊規則，使 TabPFN 的 in-context 機制直接學習「同週相位」的條件分佈。
Loss/訓練: 零樣本推理，無額外微調 Loss；TabPFN 內部基於 NLL 預訓練，推理時直接輸出 mean。

## §3 · 數據層
資料規模/頻率/市場/時段: 使用 monash_tourism_monthly (月頻) 與 m4_hourly (小時頻) 公開數據集；僅選取 item id 0 與 1 進行驗證。
怎麼來: 直接沿用 TabPFN-TS 官方 Colab 腳本數據劃分。
樣本外與容量假設: 預測長度設為週期 1 倍或 2 倍，歷史長度為週期 1 到 4 倍；假設數據週期在樣本外保持穩定，未驗證跨 regime 或週期漂移情境。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations:** 仍處初步階段；參數 k 僅憑經驗設定；未納入天氣等協變量；未來需更廣泛數據集驗證。
**6.2 推斷的隱含假設:**
- Regime 依賴: 假設週期結構在樣本外穩定，未處理週期漂移或結構性斷點。
- 容量/成本: 零樣本推理成本低，但回溯構建矩陣在長序列下記憶體開銷隨 lookback 線性增長。
- 數據泄漏: 若週期提取階段使用了全量數據（含預測區間），將產生嚴重前瞻偏差；導讀未明確說明 FFT 是否嚴格限制於訓練視窗。
- Survivorship: 僅測試公開數據集特定 item，未驗證存活者偏差或市場微結構影響。

**7.1 可證偽預測帶日期:** 若於 2025-12-31 前在含週期漂移的真實量價數據（如加密貨幣小時頻）上進行滾動回測，本法 MAE 將劣於動態週期檢測基線，因單一 FFT 主導週期無法適配 regime 切換。

## §8 · For the Reader
- **因子研究員**: 將此特徵構建邏輯嵌入因子挖掘流水線，用 FFT 週期替代固定視窗，提升週期性因子的樣本外穩定性。
- **高頻執行**: 不適用。本法回溯切片與表格推理延遲高，且未建模訂單簿微結構與滑點。
- **組合配置**: 可作為中低頻資產輪動的週期濾波器，但需結合風險預算與交易成本模型，避免週期失效時的過度暴露。
- **LLM-agent / RL 策略**: 可將 FFT 週期作為環境狀態觀測的一部分，供 RL 智能體學習週期切換策略；或作為 Agent 的时序特徵預處理器。
- **研究學生**: 重點跟進 k 值動態選擇與多週期疊加特徵構建，這是突破單一主導週期瓶頸的關鍵路徑。
