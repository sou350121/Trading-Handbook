## §1 · 架構 / Core Mechanism
| 改動維度 | 前作（兩步解耦/靜態蒸餾） | ADD 本方法 |
|:---|:---|:---|
| 訓練範式 | 兩步分離訓練，梯度斷裂 | 端到端同步優化，解碼器重建約束 |
| 去噪機制 | 固定權重蒸餾或純對抗 | 動態自蒸餾（日 IC + 樣本 MSE 雙重權重） |
| 數據增強 | 時序滑動/加噪/Dropout | 解碼器驅動的跨日特徵混合（β/α 交換） |

⚡ **Eureka:** 用解碼器保信息純度，再以動態 IC 權重蒸餾殘餘噪聲，讓「分離」與「增強」在同一計算圖內閉環。

```text
Input X (360d) 
  ├─> [Enc_E] ──> f_E (超額特徵) ──> [Pre_E] / [Adv_E]
  └─> [Enc_M] ──> f_M (市場特徵) ──> [Pre_M] / [Adv_M]
          │
          └─> [Decoder] ──> X̂ (重建約束)
          │
          └─> [Dynamic Self-Distill] (日/樣本權重重加權)
                └─> Output (超額回歸 + 市場分類)
```

## §2 · 數學層
📌 **Napkin Formula:**
`L_total = L_ret(E/M) + λ_adv·L_adv + L_rec + L_distill(α·w_day·w_sample)`
複雜度：前向 `O(T·N·d)`，GRU(64)+MLP 輕量，單卡可跑。
直覺：對抗項強行正交化 β/α 空間，蒸餾項按日/樣本表現動態重加權（表現差則權重↑），解碼器防止特徵坍縮。
Loss/訓練：Adam (lr=0.001, BS=5000)，迭代至收斂；教師/學生同構，超額用 MSE，市場用 CrossEntropy。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations**：訓練策略可進一步簡化；解耦結構設計仍有優化空間；未討論極端行情（如流動性枯竭、政策突變）下的表征穩定性。
**6.2 推斷的隱含假設**：
- **Regime 依賴**：動態權重 `w_day` 依賴歷史 IC 極值，若市場分佈偏移（如註冊制後波動率結構改變），權重可能反向放大噪聲。
- **容量/成本**：日頻聚合已丟失盤口微結構，實盤容量受限；回測未計滑點/手續費/漲跌停限制。
- **數據泄漏**：跨日特徵混合（`f_E^p + f_M^q`）若未嚴格按交易日切分，可能引入未來信息。
- **Survivorship**：未明確說明退市/長期停牌股票的剔除邏輯，測試集可能存在生存偏差。

- ❌ 錯答：混淆對抗損失（壓降跨模態信息）與蒸餾損失（提煉同模態知識）的優化目標，或忽略跨日增強的時間嚴謹性。

## §8 · For the Reader
- **因子研究員**：關注解碼器重建約束如何替代傳統正交化/中性化流程，可嘗試將 `L_rec` 替換為行業/市值暴露約束。
- **高頻執行**：不適用。日頻滑動窗口已過濾微結構，需重構為盤口級表征模塊。
- **組合配置**：可用於 β/α 權重動態調整或風險預算分配，但必須疊加 Covariance 估計與交易成本模型。
- **LLM-agent / RL 策略**：可作為狀態表征預訓練模塊（State Encoder），但需重構 Reward 函數以匹配動態蒸餾的權重反饋機制。
