> **核心定位**：將通用時間序列基礎模型（TimesFM）透過持續預訓練與對數損失/掩碼策略，遷移至金融價格預測。解了「通用時序模型直接套用價格數據會因尺度差異與崩盤事件導致 NaN 與失效」的 prior gap。

**五軸座標**

| 數據模態 | 時間尺度 | 學習範式 | Alpha機制 | 人機協作 |
|:-:|:-:|:-:|:-:|:-:|
| `量价表格` | `日频波段` | `监督回归` | `端到端表征` | `全自动黑盒` |

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 將 2 億參數的僅解碼器時序基礎模型 TimesFM 持續預訓練至金融價格數據。② 核心 trick 為對數變換 MSE 損失（穩定崩盤與跨資產尺度）與隨機上下文掩碼增強。③ 對「端到端表徵」軸★，證明基礎模型無需特徵工程即可透過損失函數重構適應金融非平穩性。④ 導讀未給量化結果（表格數據均為圖片未披露，僅提及「高達 10% 的最大收益」與「訓練損失降至原始 70%」）。

**X-Ray.** 本方法將五軸中的「端到端表徵」推向極致：放棄傳統因子工程，直接讓模型從原始價格序列中學習隱式動態。它解了兩個舊工程坑：一是跨資產尺度差異導致的梯度失衡，二是極端行情引發的 NaN 崩潰。對數損失本質上將絕對誤差轉為相對誤差，掩碼策略則強迫模型學習多尺度依賴而非記憶固定視窗。預測其打不開的 envelope 在於：該架構仍屬單變量/逐資產預測，未內建橫截面相對價值邏輯，且模擬交易假設零成本與固定持有期，未處理流動性衝擊與訂單簿微結構。對量化讀者的意義在於提供了一個「低特徵工程開銷、高算力換取表徵」的 baseline，適合用於多資產動量/均值回歸的初始信號生成，但需疊加風險模型與成本過濾。

## §1 · 架構 / Core Mechanism
1.1 三大改動 vs 前作
| 改動維度 | 原始 TimesFM | 金融微調版 | 工程意圖 |
|---|---|---|---|
| 損失函數 | 原始 MSE | 對數變換 MSE (`log(y)`) | 消除資產價格尺度差異，抑制崩盤 NaN |
| 上下文策略 | 固定/標準掩碼 | 隨機 `tstart`/`tend` 掩碼增強 | 防止過擬合固定視窗，適應多頻率數據 |
| 訓練數據 | 通用時序/合成數據 | 純金融價格（股/指/外匯/加密） | 領域遷移，消除分佈偏移 |

1.2 ⚡ Eureka 一句話 trick + 直覺
對數損失將「絕對價格誤差」映射為「百分比誤差」，使模型在訓練時自動完成跨資產的標準化，無需手動 Z-score 或收益率轉換。

1.3 信息流 ASCII 圖
```
[原始價格序列 y] -> log(y) -> [隨機掩碼切片] -> [TimesFM Decoder] -> [預測 log(y_hat)]
      ^                                                      |
      |______________________________________________________| (Loss: MSE on log space)
```

## §2 · 數學層
📌 Napkin Formula：
`L = MSE(log(y_input), log(y_target))`
直覺：對數變換壓縮大數值區間，放大小數值區間的相對變化。當價格暴跌時，`log(y)` 的變化率遠低於 `y` 本身，梯度不會爆炸。
Loss/訓練細節：SGD + 線性預熱 + 餘弦衰減，峰值 LR `5e-4`。訓練損失穩定在原始值的 `70%` 左右。

## §3 · 數據層
規模/頻率：`100K` 個時間序列，`90M` 個時間點。頻率涵蓋小時與日級。市場：股票、指數、外匯、加密貨幣。來源：Yahoo Finance, Binance API。時段：訓練/驗證集為 `2023` 年 `1` 月 `1` 日之前；測試集為 `2023` 年以後（嚴格防範前瞻偏差）。容量假設：模型僅處理單變量序列，未使用橫截面數據；訓練未使用合成數據或重採樣。

## §6 · 失效與隱含假設
6.1 論文自述 limitations：訓練超過 `100` 個週期或更大 LR 會出現過擬合；建議未來使用更大數據集、更強增強或早停。未處理橫截面依賴。
6.2 推斷隱含假設：
- Regime 依賴：對數損失假設價格變動服從幾何布朗運動或至少對數平穩，在結構性斷裂（如退市、合併）時可能失效。
- 容量/成本：模擬策略限制每日 `l1` 范數不超過 `1`，假設無限流動性與零滑點。實盤中長持有期會暴露於宏觀風險。
- 數據泄漏：訓練/驗證集同屬 `2023` 年前，市場趨勢高度相關，模型可能記憶週期性而非學習因果。
- Survivorship：使用公開 API 數據，通常包含已退市資產，若未處理則存在生存偏差。

## §8 · For the Reader
- **因子研究員**：可直接將 `log(y)` 預測值轉為未來 `h` 期收益率信號，疊加傳統動量因子做正交化測試。注意掩碼策略產生的多尺度輸出可構建頻率分解因子。
- **高頻執行**：此模型輸出為日/小時級，不適用 HFT。但可用作盤前倉位預警，過濾低勝率時段。實盤需嚴格加入成本模型，否則名義收益會被滑點吞噬。
- **組合配置**：將各資產預測收益輸入 Black-Litterman 或風險平價模型。注意模型未提供預測區間（Uncertainty），需自行計算殘差標準差作為置信權重。
- **LLM-agent / RL 策略**：可將 TimesFM 的隱狀態（Hidden States）作為環境觀測輸入 RL 代理，替代人工特徵。需解決獎勵函數與交易成本的非凸性。
- **研究學生**：復現重點不在架構，而在數據管道與損失重構。嘗試替換為 `Huber Loss` 或 `Quantile Loss` 對比對數 MSE 的魯棒性，是極佳的畢業設計起點。
