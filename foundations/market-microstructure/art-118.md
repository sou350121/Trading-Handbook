# 揭秘高频交易“军备竞赛” 解構（揭秘高频交易“军备竞赛”）

> **發布**：2024-12-11 · （無 venue）
> **QuantML 導讀**：[揭秘高频交易“军备竞赛”](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247488330&idx=1&sn=963b5ac770f2ce7c689f91881da4f2ef&chksm=ce7e7454f909fd428b16bb1711ae1914ef05b782bce9459f375f6e6471c82096267dfb988f70#rd)
> **核心定位**：落點於微观盘口數據模態與高频日内時間尺度，以监督回归範式解構因子挖掘機制。補足傳統LOB數據無法觀測「失敗競爭」的Prior Gap，將隱性速度優勢轉化為可計量的流動性成本指標。

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** 本文突破傳統限價訂單簿（LOB）僅記錄成功訂單的局限，引入交易所全量微秒級消息數據（含失敗指令），精準識別高頻延遲套利（Latency Arbitrage）競爭事件。核心 Trick 在於以「信息視野（Information Horizon）」為窗口，結合入站/出站時間戳重構訂單簿狀態，將速度競爭轉化為可統計的市場事件。此方法對微观盘口數據軸★，直接量化了速度優勢對流動性成本的實質侵蝕。關鍵實證數字：延遲套利稅為0.42个基点（基於總交易量）。

**X-Ray.** 本文在五軸 Pareto 上明確偏向數據模態升級與可解釋性，解了傳統回測無法區分「主動搶跑」與「被動成交」的工程坑：過去實盤團隊只能透過價差擴大或滑點異常推測速度摩擦，現在可直接以失敗消息反證競爭存在。然而，該框架未打開跨市場/跨資產的 envelope，其微秒級時間同步假設高度依賴單一交易所（LSE）的硬體時鐘架構與特定撮合邏輯，未計入真實報單/撤單的網絡延遲與排隊優先級（Price-Time vs Pro-rata）。對量化讀者而言，此文非提供可直接交易的 Alpha，而是提供了「流動性成本拆解」的基準錨點；實盤執行演算法可藉此校準滑點模型，避免將速度競爭導致的價差擴大誤判為因子失效或市場衝擊。

## §1 · 架構 / Core Mechanism
| 維度 | 傳統 LOB 研究 | 本文架構 |
|---|---|---|
| 數據源 | 僅記錄成功成交/掛單快照 | 交易所全量消息流（含失敗指令/撤單過期） |
| 事件定義 | 單一價格/成交量觸發 | 多參與者 + 時間同步(<500微秒) + 成功/失敗結果對比 |
| 成本度量 | 逆向選擇/有效價差 | 延遲套利稅（Latency Arbitrage Tax） |

⚡ **Eureka:** 用「失敗消息」反證競爭存在。傳統數據只看贏家，本文透過「取消太晚」或「市場訂單未能執行」的錯誤回傳，直接鎖定輸家，從而重構完整的軍備競賽圖譜。
**信息流 ASCII:**
```
參與者A/B 報單/撤單 -> 交易所外牆時間戳(微秒) -> 匹配引擎撮合
      -> 成功回傳(成交/掛單) / 失敗回傳(過期/太晚)
      -> 事件分類器(比對M1/M2間隔<信息視野) -> 流動性成本聚合
```

## §2 · 數學層
📌 **Napkin Formula:**
`延迟套利收益 = 0.3354 * 交易量 + 0.0066 * 波动性 * 平均交易量`
**複雜度:** O(N) 線性回歸（N為股票-日樣本）。
**直覺:** 將微觀事件收益宏觀化，假設套利規模與市場活躍度及波動呈線性耦合。微秒級競爭的總盤子不依賴單一因子，而是由流動性深度與價格不確定性共同驅動。
**Loss/訓練細節:** 非深度學習框架，屬實證計量經濟學設定。以 OLS 擬合跨期收益與宏觀流動性指標的線性關係，無反向傳播、正則化或超參數搜尋細節；模型穩健性依賴敏感性分析（調整時間窗口/參與者門檻）。

## §3 · 數據層
- **規模/頻率:** 約22億條消息，每個股票-日期平均約15萬條。時間戳精度微秒級（硬體時鐘100纳秒）。
- **市場/時段:** 倫敦證券交易所（LSE），2015年8月17日至10月16日（9周），涵蓋FTSE 350指數所有股票。
- **來源/處理:** LSE 商業數據。剔除9月7日數據損壞日。出站消息維護LOB狀態，入站/出站ID交叉比對分類市場事件。
- **樣本外與容量:** 模型外推至2014-2018英國市場及2018/2020全球市場，假設微秒級競爭結構與稅率在不同年份保持穩定（未驗證）。真實容量假設未披露，但數據顯示前6名參與者主導82%比賽，隱含極高門檻。

**解讀:** 前SOTA欄全數「未披露」，因傳統LOB研究無法直接計算此指標，本文屬首次量化。Δ欄依紀律留白。0.42 bps 為基於總交易量的稅率，若僅計非延遲套利交易量則為0.53 bps。此數值反映的是市場結構性摩擦成本，非策略可捕獲的淨Alpha；實盤中若未扣除執行延遲與交易所費用，直接套用此收益預期將導致嚴重過擬合。部分 Δ 可能源於樣本僅9周且聚焦單一交易所，跨市場泛化能力待驗證。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations:** 數據僅限LSE與FTSE 350；時間窗口設定（500微秒）具主觀性；部分事件收益為負（約20%），顯示存在噪音信號；未覆蓋其他資產類別。
**6.2 推斷的隱含假設:** 
- **Regime 依賴:** 高度依賴微秒級競爭環境與現有撮合邏輯。若交易所改為批量拍賣或引入隨機延遲，結構將失效。
- **容量/成本:** 前6名參與者主導82%比賽，隱含極高技術與資本門檻；未計入真實網絡延遲、報單費與稅費。
- **數據泄漏:** 純歷史消息重構，泄漏風險低；但樣本僅9周，跨期穩定性未驗證。

## §8 · For the Reader
- **因子研究員:** 將延遲套利稅納入交易成本模型（TCM），避免將速度摩擦導致的價差擴大誤判為因子衰減或市場衝擊。
- **高頻執行:** 優化訂單路由與報單時機，針對前6名參與者主導的時段降低主動成交比例，改用被動掛單或冰山單規避競爭。
- **組合配置:** 在波動率放大且交易量激增的Regime下，調高流動性成本預算，避免過度頻繁調倉；可將延遲套利稅作為動態交易費率的輸入變數。
