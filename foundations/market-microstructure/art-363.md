# 金融基础模型 解構

> **發布**：2026-01-17 · ICML25
> **QuantML 導讀**：[HRT | 金融基础模型与 Scaling Laws](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247493015&idx=1&sn=2d3cc7061cbc6dbd68ad98d7913fbc13&chksm=ce7d8289f90a0b9f6c89714d728a1a581b7f6c8fc6f77b9fe70fbdd1258702ce8429c70a6fd1#rd)

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 將 NLP/CV 的 Scaling Laws 驗證並遷移至金融微觀結構，證明參數量與數據量增加可穩定提升 LOB 預測能力。② 核心 trick 為 Event-based Tokenization 配合 Transformer/RNN 的 Smart-Fast 混合架構，在 tick-to-tick 場景下動態權衡推理延遲與上下文捕捉。③ 對「高频日内」軸而言，它直接重構了特徵工程與模型複雜度的資源分配邏輯。④ 導讀未給量化結果。

**X-Ray.** 直指量化底層的「算力-延遲-信噪比」不可能三角。傳統 LOB 策略依賴人工特徵與淺層樹模型，雖能壓低延遲，但無法捕捉非線性流動性枯竭與逆向選擇的動態關聯。本法將市場視為非均勻事件流，以 Event-based Tokenization 替代 Time-based 切片，保留微觀結構的 Burst 特徵，這在低信噪比環境中是必要的歸納偏置鬆綁。然而，生成式大模型在金融場景的真實 envelope 並未打開：缺乏完美 Simulator 意味著端到端 RL 無法直接閉環，模型仍停留在「預測器」而非「執行器」階段。對實盤研究員而言，此架構的價值不在於直接替換現有 Alpha，而在於提供一套可擴展的表征學習基座；其失效風險在於非平穩性下的分佈外推與硬體成本邊際遞減，若無法將推理延遲壓至微秒級，Smart 的優勢將被滑點與對手方反應完全吞噬。

## §1 · 架構 / Core Mechanism
**1.1 三大改動 vs 前作**
| 維度 | 前作 (Time-based / Single Arch) | 本法 (Event-based / Hybrid) |
|---|---|---|
| Tokenization | 時間切片 (分鐘/秒)，平滑微觀細節 | 事件流/Tick 觸發，保留 Burst 與流動性意圖 |
| 架構選擇 | 單一 Transformer 或 RNN | Transformer (長上下文) + RNN/SSM (O(1) 低延遲) 動態路由 |
| 特徵工程 | 依賴人工價量因子與領域知識 | 端到端表征學習，原始 LOB 流直接映射至向量空間 |

**1.2 ⚡ Eureka 一句話 trick**
事件流 Tokenization + Smart-Fast 動態路由，在低信噪比環境下重構算力與延遲的 Pareto 邊界。

**1.3 信息流 ASCII**
```
[LOB Raw Stream] → [Event Tokenizer] → [Transformer Branch (Long Context)] 
                                      ↘ [Router/Selector] → [Prediction Head] → [Signal]
                                      ↗ [RNN/SSM Branch (O(1) Latency)]
```

## §2 · 數學層
📌 **Napkin Formula**
```
P(y_t | x_{1:t}) = f_θ(Tokenizer(Event_t))
Complexity: O(1) [RNN] vs O(L) or O(L^2) [Transformer]
L_total = L_pred + λ·Latency(model_arch)
```
**直覺**：以事件觸發替代時間切片，將非均勻市場活動對齊至序列空間；透過架構混合在預測精度與推理延遲間尋找最優解。複雜度直接決定能否跨過實盤閾值。
**Loss/訓練細節**：導讀未披露具體 Loss 函數與訓練超參，僅提及基於 100TB 數據與 GPU 集群進行大規模預訓練。

## §3 · 數據層
- **規模/頻率**：超過 100TB 清洗後市場數據（涵蓋全球全資產類別，相當於數萬億 Tokens）；微觀盤口（LOB）級別，事件流/Tick 頻率。
- **來源**：HRT 內部歷史流水與訂單簿快照。
- **樣本外與容量假設**：依賴市場微觀結構的穩定性，但導讀明確指出市場為非平穩非零和博弈，容量與數據泄漏風險未披露。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations**
- 無完美 Simulator，回測無法完美複刻市場衝擊與對手方反應。
- 極低信噪比導致基線必須達 Super-human 水平。
- 長週期預測有效樣本量急劇下降且易過擬合。

**6.2 推斷的隱含假設**
- **Regime 依賴**：假設事件流 Tokenization 能穩定捕捉流動性意圖，分佈在訓練與實盤間保持足夠相似性。
- **成本/延遲**：隱含假設硬體算力擴張能持續壓降推理延遲，否則 Smart 優勢被滑點吞噬。
- **數據泄漏**：未提及跨市場/跨資產的樣本劃分策略，高頻場景下極易引入前瞻偏差。

| 同軸對手 | 關鍵差異軸 | Open? | Status |
|---|---|---|---|
| 傳統 LOB 淺層模型 (XGBoost/NN) | 特徵工程 vs 端到端表征 / 時間切片 vs 事件流 | 部分開源 | 成熟但維度受限 |
| 純 Transformer 預測器 | 長上下文捕捉 vs 推理延遲 O(L^2) | 研究階段 | 延遲瓶頸明顯 |
| 本方法 (HRT Foundation) | Smart-Fast 動態路由 / 生成式預訓練 | 閉源 | 內部驗證中 |

🎤 **Interview Tip**
- **正確答**：「金融基礎模型的核心不在於替換特徵工程，而在於透過事件流 Tokenization 與架構混合，在低信噪比環境下重構算力與延遲的 Pareto 邊界；其實盤價值取決於能否將推理延遲壓至微秒級並解決非平穩性下的 OOD 泛化。」
- **錯答**：「大模型參數量越大，Sharpe 越高，直接替換現有因子即可上線。」

**7.1 可證偽預測**
若 2026-Q3 前無頭部做市商公開基於 Event-based Tokenization 的微秒級推理架構，或 Scaling Laws 在 LOB 數據上的收益曲線出現明顯飽和，則本法將被降維為離線特徵提取器而非端到端黑盒。

## §8 · For the Reader
- **因子研究員**：將 Event Tokenizer 視為非線性特徵提取器，替代傳統價量因子，但需警惕分佈偏移與過擬合。
- **高頻執行**：關注 Smart-Fast 路由的硬體映射，若推理延遲 > 微秒級，預測優勢將被滑點與逆向選擇完全抵消。
- **組合配置**：本法目前僅提供預測信號，未閉環至執行與風險控制，需搭配獨立的組合優化層與動態倉位管理。
- **LLM-agent/RL 策略**：無完美 Simulator 是端到端 RL 的死穴，建議先以監督學習預訓練表征，再以離線 RL 微調，避免在線探索的災難性回撤。
