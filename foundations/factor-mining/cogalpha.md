> **核心定位**：落點於「生成式大模型 × Agent自主演进」軸，將 LLM 的邏輯推理與遺傳演算法的代碼級變異結合，解決傳統 GP/深度學習在因子挖掘中「黑箱化」與「搜索空間狹窄」的 prior gap。

|:-:|:-:|:-:|:-:|:-:|

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 以七層 Agent 階層拆解 Alpha 挖掘任務，結合 LLM 認知推理與代碼級遺傳變異/交叉。② 核心 trick 是將 LLM 置於代碼空間執行類遺傳演算法優化，並由多 Agent 質量守門人過濾偽相關與前瞻偏差。③ 對「因子挖掘」軸而言，它將符號表達的解釋性與神經網路的非線性捕捉力統一，避免傳統 GP 的冗餘公式。④ 導讀給出完整架構在 CSI300 測試集上達成 IC 0.0591 與 RankIC 0.0814。

**X-Ray.** 本框架在五軸 Pareto 上明確選擇「可解釋性 × 搜索廣度」，犧牲部分訓練速度換取邏輯透明。它解決了傳統 GP 生成冗餘數學式、DL 模型遭遇 Regime Shift 時脆弱的工程坑，並透過多 Agent 守門機制阻斷代碼級的前瞻偏差。然而，其 envelope 打不開的地方在於：代碼空間的變異交叉高度依賴 LLM 的上下文窗口與推理延遲，實盤迭代成本與算力門檻未披露；且日頻波段因子在流動性枯竭或極端波動下的容量假設仍屬黑箱。對量化讀者而言，此架構提供了一套「邏輯可審計」的因子生產線，但需警惕 IC 提升是否源於數據分佈的過擬合，而非真正的經濟學因果。

## §1 · 架構 / Core Mechanism
| 改動維度 | 前作 (傳統 GP / 純 LLM) | CogAlpha 改動 | 工程意圖 |
|---|---|---|---|
| 搜索空間 | 符號樹堆疊或提示詞直出 | 七層代理階層 + 代碼級變異/交叉 | 打破單一邏輯同質化，擴展結構創新 |
| 質量控制 | 回測績效單維過濾 | 多 Agent 守門人 (Code/Repair/Judge/Logic) | 剔除偽相關與語法/運行時 Bug，強迫邏輯自洽 |
| 優化機制 | 靜態訓練或一次性生成 | 思維進化 (Thinking Evolution) 迭代池 | 模擬人類研究員的假設-驗證-重構循環 |

⚡ **Eureka:** 將 LLM 的推理能力錨定在 `Python/Pandas` 代碼空間，以代碼片段為「基因」執行遺傳演算法，使因子從「統計擬合」升級為「邏輯演進」。
**信息流:**
```
[OHLCV] → Level I~VII Agents (Prompt/Paraphrase) → LLM Code Gen
   ↓
[Multi-Agent Checker] → (Syntax/Logic/Look-ahead) → Pass/Fail
   ↓
[Evolution Pool] → Mutation / Crossover → Next Gen → [IC/RankIC Eval]
```

## §2 · 數學層
📌 **Napkin Formula:** 無傳統梯度 Loss。適應度評估直接錨定 $IC(f_{code}, r_{10d})$ 與 $RankIC$，優化目標為離散代碼空間的搜索。複雜度：代碼變異/交叉為組合優化問題，LLM 推理引入 $O(N \cdot L)$ 延遲。直覺：以代碼邏輯為基因，透過多 Agent 守門人過濾語法/邏輯錯誤，僅保留經濟學意義自洽的變異體進入下一代。訓練細節：無 backprop，依賴 IC/RankIC 作為強化信號，配合消融實驗中的 RankICIR 遞增驗證組件貢獻。

## §5 · 評測 / Benchmark
| 數據集/市場 | Metric | 前SOTA/基線 | 本方法 | Δ |
|---|---|---|---|---|
| CSI300 (2021-2024) | IC | LightGBM 0.0269 | CogAlpha 0.0591 | +0.0322 |
| CSI300 (2021-2024) | IC | Transformer -0.0090 | CogAlpha 0.0591 | +0.0681 |
| CSI300 (2021-2024) | IC | GPT-4.1 0.0118 | CogAlpha 0.0591 | +0.0473 |
| CSI300 (2021-2024) | IC | GPT-OSS-120B 0.03 | CogAlpha 0.0591 | +0.0291 |
| CSI300 (2021-2024) | RankIC | 未披露基線 | CogAlpha 0.0814 | 未披露 |
| CSI300 (2021-2024) | RankICIR | Agent 0.2595 → Full 0.4350 | CogAlpha 0.4350 | +0.1755 (vs 基礎Agent) |

**解讀:** Δ 在 IC 上的提升主要來自代碼級變異對非線性特徵的捕捉，以及多 Agent 守門人過濾了 DL 模型的偽相關。但 RankIC 與 RankICIR 的基線未完整披露，Δ 的絕對值需謹慎解讀。案例中因子從 IC 0.0090 進化至 0.0141，顯示迭代有效，但未計入交易成本與換手頻率，實盤 Δ 可能因摩擦成本大幅縮水。此外，測試集僅至 2024 年，未驗證極端 Regime 下的前瞻性偏差控制是否徹底。

## §6 · 失效與隱含假設
6.1 論文自述 limitations：未明確列出 limitations 段落，僅提及未來需擴展至另類數據與跨資產類別。
6.2 推斷的隱含假設：
- Regime 依賴：七層結構中的 `AgentRegimeGating` 假設市場狀態可被穩定分類，但在快速切換的流動性危機中可能滯後。
- 容量/成本：日頻波段因子假設 CSI300 具備足夠深度消化信號，未披露單因子容量上限與實盤滑點模型。
- 數據泄漏：守門人雖排查 Look-ahead Bias，但代碼級變異若未嚴格限制時間窗口，仍可能隱性引入未來信息。
- 算力假設：多輪 LLM 推理與代碼執行依賴高並發 GPU/TPU 集群，中小機構複現成本極高。

## §7 · 對比 & 面試 Tip
| 同軸對手 | 關鍵差異軸 | Open? | Status |
|---|---|---|---|
| 傳統 GP (如 AlphaGen) | 符號樹 vs 代碼級進化 | 部分開源 | 成熟但搜索空間窄 |
| 純 LLM 因子生成 | 提示詞直出 vs 多 Agent 守門+進化 | 閉源/實驗性 | 易過擬合，邏輯不可審計 |
| 深度學習因子 (如 Transformer) | 黑箱擬合 vs 可解釋代碼 | 開源 | 預測力強但 Regime 脆弱 |

🎤 **Interview Tip:**
- ✅ 正確答：「CogAlpha 的核心不在於模型架構，而在於將因子生成從『參數優化』轉為『代碼空間的離散搜索』。多 Agent 守門人解決了傳統 GP 的語法冗餘與 DL 的邏輯黑箱，但實盤落地需補齊換手成本與容量壓力測試。」
- ❌ 錯答：「它只是用 GPT 寫 Python 腳本，IC 高是因為訓練數據多。」（忽略代碼級遺傳機制與守門人過濾的結構創新）
7.1 可證偽預測：若未來實盤驗證顯示 RankICIR 顯著低於導讀給出的 0.4350，或代碼級變異未能通過嚴格的前瞻性偏差壓力測試，則框架的「認知進化」優勢可能僅限於離線回測環境。

## §8 · For the Reader
- **因子研究員**：直接複用七層階層的分類邏輯，將你的手工因子映射至 Level I~VII，檢查是否覆蓋了「幾何融合」與「體制門控」盲區。
- **高頻執行/組合配置**：警惕日頻因子的換手頻率。代碼級變異可能生成高頻信號，需加入 turnover 約束與成本模型，否則 IC 提升會被滑點吞噬。
- **LLM-Agent/RL 策略**：關注 `Thinking Evolution` 的獎勵設計。IC/RankIC 作為稀疏獎勵，可嘗試結合 PPO 或 DPO 優化 LLM 的代碼生成策略，減少無效變異。
