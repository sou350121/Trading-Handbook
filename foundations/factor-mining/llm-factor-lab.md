# LLM Factor Lab 解構（LLM Factor Lab）

> **發布**：2026-07-18 · （無 venue）
> **QuantML 導讀**：[GOOGLE ｜ 大模型因子挖掘的真实生存率](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247494295&idx=1&sn=24f461ebdbbc6e609e46875cac136c9c&chksm=ce7d8d89f90a049f23d699f03afd771ae7a45d50ee5467b6dad7ea2cd412b433b1de26099f19#rd)
> **核心定位**：將生成式大模型置於預設會計語法空間內進行多輪反思迭代，以「人設實驗框架 + AI內生演化」取代傳統黑箱搜索；解決因子研發中「過度擬合歷史噪音」與「已知風險因子吸收」的工程斷層。

|:-:|:-:|:-:|:-:|:-:|

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** 用LLM在预设会计公式空间内自主演化因子，经严苛学术检验仅极少数存活。核心 trick 是人设语法与回测管线，LLM多轮反思迭代；引入多重检验、因子吸收与异象赛马的工业化过滤流水线。這對因子挖掘軸★的意義在於將「人工試錯」轉為「結構化搜索+嚴苛過濾」。關鍵實證數字：280個候選中僅數個存活，FF5下僅8個保留顯著Alpha。

**X-Ray.** 本方法將因子挖掘的 Pareto 前沿從「搜索空間大小」轉移至「過濾管線嚴苛度」。傳統 GenProg/ML 的失效模式是黑箱公式在低信噪比環境中快速擬合噪音，且缺乏經濟學可解釋性；本架構透過限定 66 個會計變量與 24 個操作符的語法空間，強制 LLM 在可解釋的結構內重組。它打不開的 envelope 是「會計數據本身的定價邊界」：市場已被 FF5/q-factor/異象庫重度佔領，LLM 無法憑空創造新風險溢價，多數高 t 值公式本質是舊酒裝新瓶。對量化讀者的真實價值不在 LLM 的參數量，而在於「多輪反思+跨檢驗過濾」的工業化流水線設計；這套框架可直接遷移至因子庫維護、異象壽命追蹤與組合正交化預處理。

## §1 · 架構 / Core Mechanism
| 維度 | 傳統 GenProg / 純數值挖掘 | LLM Factor Lab |
|---|---|---|
| 搜索空間 | 開放數學表達式，易生成極長黑箱公式 | 預設會計語法空間（66變量 + 24操作符），結構受限且可解釋 |
| 迭代機制 | 單次適應度評估或固定交叉變異 | 多輪反思迭代（提出假設-回測評估-反思），保留前代思維鏈上下文 |
| 驗證管線 | 單一歷史回測 Sharpe/Alpha | 工業化過濾流水線（多重檢驗修正 → 因子吸收 → 異象賽馬 → 樣本外穩定性） |

⚡ **Eureka Trick:** 人設語法約束 + 多輪反思迭代。直覺：限制表達式自由度可避免組合爆炸與過擬合，強制模型在「存量vs流量」「結構vs時機」等經濟學維度上進行邏輯重組，而非盲目湊數學項。

```
[人類定義語法/變量庫] → [LLM Gen 0 生成候選] → [標準化回測管線]
       ↑                                      ↓
[反思迭代/思維鏈傳遞] ← [統計檢驗/過濾] ← [月度截面/組合構建]
       ↓
[獨立生存者] → [多重檢驗/因子吸收/異象賽馬] → [終極幸存者]
```

## §2 · 數學層
📌 **Napkin Formula:** `NEG(MUL(RATIO(ADD(dltt, dlc), at), GROWTH(ppent, 1)))`
**複雜度:** O(G × N × T)，G為代數（7代），N為單代候選數（5個），T為回測與檢驗計算量。非梯度優化，依賴離散適應度（|t|/Sharpe）進行進化選擇。
**直覺:** 公式強制引入「存量槓桿」約束「流量投資增速」，將資產負債表脆弱性與管理層冒險行為結構化。訓練細節無傳統 loss，而是以 Fama-MacBeth |t| > 1.96 與符號預期為適應度閾值，透過 Stepwise backward elimination 剔除秩相關 > 0.95 的冗餘信號。

## §3 · 數據層
- **資料規模/頻率:** Compustat 年度會計變量（66個基礎科目）匹配 CRSP 交易數據，轉換為月度截面。
- **市場/時段:** 樣本區間橫跨 1963年7月至2024年12月。
- **處理邏輯:** 嚴格保留至少 6 個月的延遲以防前瞻偏差；每月按因子值進行五分組（Quintile sorts），利用 NYSE 單一斷點，剔除 NYSE 市值後 10% 的微盤股，計算市值加權超額收益與多空組合（Q5 - Q1）。
- **樣本外假設:** 假設會計數據披露與市場定價機制在長週期內結構穩定；post-2010 子樣本用於檢驗衰減趨勢。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations:** 大模型無法憑空製造全新超額收益來源；多數高表現公式在資產定價放大鏡下屬「舊酒裝新瓶」；會計語法空間限制可能遺漏非財務數據或高頻結構的 Alpha。
**6.2 推斷的隱含假設:** 
- **Regime 依賴:** 假設會計準則與市場定價邏輯在 1963年7月至2024年12月 間未發生結構性斷裂；post-2010 衰減暗示微觀結構或因子擁擠度變化會削弱信號。
- **容量/成本:** 未計入交易成本與滑點；多空組合（Q5 - Q1）在實盤中受融券成本與流動性約束。
- **數據泄漏/前瞻:** 雖保留 6 個月延遲，但 Compustat 數據重述與 CRSP 復權處理若未嚴格對齊，仍可能引入隱性前瞻。
- **Survivorship:** 剔除 NYSE 市值後 10% 可緩解微盤股效應，但未明確處理退市/破產樣本的生存偏差。

| 同軸對手 | 關鍵差異軸 | Open? | Status |

| RL for Factor Mining | 強化學習策略搜索 vs LLM 反思迭代 | 部分開源 | 實驗階段，獎勵函數設計難 |

🎤 **Interview Tip:** 
- **正確答:** 因子挖掘的瓶頸不在搜索能力，而在過濾管線。LLM 的重組效率極高，但必須透過多重檢驗修正、已知因子吸收（FF5/q-factor）與異象賽馬（Chen-Zimmermann）剝離水分；實盤需計入成本與容量約束。
- **錯答:** 只要 LLM 參數夠大或推理深度夠深，就能自動發現全新 Alpha；單一歷史回測 Sharpe 高即可上線。

**7.1 可證偽預測帶日期:** 若於 2026-12-31 前將此過濾流水線遷移至 A 股或港股會計數據，預期獨立生存者比例將低於 5%，主因新興市場會計披露質量與定價效率差異導致因子吸收率更高。

## §8 · For the Reader
- **因子研究員:** 將此架構視為「因子壽命追蹤器」。把現有因子庫送入 FF5/q-factor 吸收測試，識別哪些是純淨增量、哪些是風險溢價重包裝。
- **組合配置/風險:** 關注多空組合在 FF5 下的 t 值衰減路徑；實盤構建時需對高相關因子（秩相關 > 0.95）進行正交化或剔除，避免隱性重疊暴露。
- **LLM-Agent / 量化開發:** 重點複現「反思迭代+思維鏈傳遞」機制。將回測統計結果結構化為 Prompt 上下文，比單純放大模型參數更能提升搜索效率。
- **研究學生:** 練習設計自帶多重過濾的實驗環境。不要追求單次回測的極端 Sharpe，優先掌握 FDR/Bonferroni 修正與條件顯著性檢驗的實作細節。
