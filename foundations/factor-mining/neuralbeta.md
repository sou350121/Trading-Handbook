**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 提出 NeuralBeta 以深度網路動態估計單/多變量 β 係數。② 核心 trick 為受正則化加權線性回歸啟發的可解釋輸出層，以樣本加權形式顯式量化數據重要性。③ 對「人機協同可解釋」軸★：將注意力機制轉化為可審計的 WLS 權重，使 regime 切換具備透明度。④ 導讀未給量化結果。

**X-Ray.** 本質是將滾動視窗的隱式等權/指數衰減，替換為可微的樣本重要性評分。它解了傳統方法在 regime shift 時的滯後坑，但 envelope 受限於「僅依賴歷史價量視窗」，無法內生處理基本面斷層或流動性枯竭。對量化讀者的意義不在於直接替換因子暴露模型，而在於提供一個可解釋的動態權重生成器，可用於風險預算調整或交易成本感知下的樣本過濾。五軸 Pareto 上，它以輕微的訓練成本（100,000 次梯度更新）換取了權重透明度，代價是仍需人工校準視窗長度與正則化強度。

## §1 · 架構 / Core Mechanism
| 改動維度 | 前作/基線 (OLS/WLS/標準NN) | NeuralBeta 改動 |
|---|---|---|
| 權重生成 | 固定滾動窗 / 隱式注意力 | 可解釋輸出層顯式輸出正權重矩陣 $W_t$ |
| 參數估計 | 封閉解或端到端黑盒 | 結合高斯先驗 $(\mu, \Sigma)$ 的 WLS 閉式解 |
| 制度適應 | 滯後或過度平滑 | 權重在 $\beta$ 跳躍點後顯著增加，跳躍點前趨近 0 |

⚡ **Eureka:** 用網路預測「哪些歷史樣本該被加權」，而非直接預測 $\beta$ 值本身，再將權重注入解析解。
📊 **信息流:**
```
[回顧窗口 X_t, Y_t] → [特徵提取層] → [權重預測頭 → W_t] → [WLS閉式解層] → [輸出 β_t]
```

## §2 · 數學層
📌 **Napkin Formula:**
$\hat{\beta}_t = (X_t^\top W_t X_t + \Sigma^{-1})^{-1} (X_t^\top W_t Y_t + \Sigma^{-1}\mu)$
複雜度：$O(T \cdot d^2)$（$T$為視窗長度，$d$為因子維度），閉式解避免迭代優化 $\beta$。
**直覺:** $W_t$ 由網路動態生成，$\mu, \Sigma$ 作為高斯先驗穩定小樣本估計。Loss 為預測 $\beta$ 與真實標籤的 MSE，訓練僅更新權重生成參數。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations:** 依賴回顧窗口內的線性假設；未討論極端流動性枯竭下的權重崩潰；超參數魯棒性僅限於「不同超參數設置下變化較小」，未給出失效閾值。
**6.2 推斷隱含假設:**
- **Regime 依賴:** 在平穩低波動期，動態加權可能引入不必要的噪聲，表現或遜於簡單 OLS。
- **容量/成本:** 日頻波段適合中低頻，未披露 breakeven 成本；若換手過高，權重頻繁跳躍將侵蝕 Alpha。
- **數據泄漏:** 視窗構建若未嚴格對齊 $t$ 時刻的公開信息，易產生前視偏差。
- **Survivorship:** S&P 500 成分股替換若未處理退市股票，測試期指標將被高估。

🎤 **Interview Tip:**
- ✅ 正確答：「NeuralBeta 的核心不是端到端回歸，而是將網路轉化為權重生成器，輸出 $W_t$ 後用 WLS 閉式解計算 $\beta$。這保證了可解釋性，並通過正則化先驗 $(\mu, \Sigma)$ 控制過擬合。」
- ❌ 錯答：「它直接用 LSTM 預測 β 值，比 OLS 準是因為深度學習能捕捉非線性。」（忽略閉式解與權重輸出層的設計意圖）
**7.1 可證偽預測:** 若 2025-12-31 前，該架構在 A 股或 crypto 日頻數據上未展現出比 Rolling WLS 更低的 MDD，則其 regime 適應性僅限於美股成熟市場結構。

## §8 · For the Reader
- **因子研究員:** 將 $W_t$ 視為動態樣本過濾器，用於構建「制度感知型」因子暴露矩陣，替代固定 decay。
- **組合配置/風控:** 監控 $W_t$ 的熵值或集中度，作為 regime 切換的早期預警指標，觸發風險預算重分配。
- **量化開發/RL:** 將閉式解層視為可微算子，嵌入端到端強化學習環境，讓 agent 學習在交易成本約束下最優的視窗權重分佈。
