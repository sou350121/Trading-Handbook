**X-Ray.** 在「圖關係」與「高頻日內」軸上，DINS 不碰價格序列，純粹解構社交互動的拓撲與時間動態。舊工程坑在於 DGE 的 NSS 預設節點角色固定且忽略時間窗口，導致訓練目標與真實金融傳播路徑錯位。DINS 的聯合採樣強行將「時間精度」與「角色互換」注入損失函數的負半空間。預測它打不開的 envelope：純社交圖嵌入無法直接對沖流動性衝擊或訂單簿微結構噪音，若無價格/量價特徵對齊，僅能作為情緒先行指標而非直接交易信號。對量化讀者意義：提供了一套可插拔的數據層正則化方案，將「負樣本難度」從隨機分佈轉為領域先驗分佈，適合做高頻情緒因子的底層特徵工程。

## §1 · 架構 / Core Mechanism
| 維度 | 前作 (Random/Historical NSS) | DINS 改動 |
|---|---|---|
| 節點角色假設 | 僅替換接收者，預設角色固定 | 隨機發送者與接收者同時替換，捕捉雙向互換 |
| 時間動態建模 | 歷史連接或隨機時間，忽略未來窗口 | 未來時間窗口負採樣，強制模型學習交互時間精度 |
| 結構特徵捕捉 | 忽略 Posting 行為或視為普通邊 | 負自環採樣，專門評估未預期 Posting 行為的發生概率 |

⚡ **Eureka:** 「把負樣本的生成邏輯從『隨機撒網』改為『按金融社交行為先驗定向爆破』，再用正樣本增強兜底分佈。」
**信息流 ASCII:**
```
正樣本批次 B -> [DINS採樣器] -> (隨機收發 / 未來時間 / 負自環)
                      |
                      v
              [正樣本增強] (注入未來邊防梯度崩塌)
                      |
                      v
              聯合訓練批次 -> DGE模型 -> 鏈接預測損失
```

## §2 · 數學層
📌 **Napkin Formula:**
$L = -\frac{1}{|B|} \sum_{(u,v,t) \in B} [\log \sigma(f(u,v,t)) + \lambda \sum_{k} \log \sigma(-f(u, v'_k, t'_k))]$
**複雜度:** 負樣本數量增加 $O(k)$，訓練時間約翻倍。
**直覺:** 損失函數的負半空間被領域知識重構，$\lambda$ 控制正樣本增強權重以平衡梯度。Loss 為標準二元交叉熵，訓練採用時間序列交叉驗證（按月劃分）。正樣本增強約束為添加數量不超過當前批次大小，避免計算開銷過大。

## §3 · 數據層
**資料規模/頻率/市場/時段:** Reddit WSB 板塊，5分鐘時間分辨率。涵蓋 GME、AMC、BB 相關互動期。
**怎麼來:** 爬取「发帖-评论」結構，移除未知用戶、排除交互稀疏月份，時間戳歸一化至5分鐘間隔。GME 數據集為517,975个节点、2,692,485条边；AMC 為134,010个节点、1,544,006条边；BB 為30,434个节点、305,349条边。
**樣本外與容量假設:** 時間序列交叉驗證（前一個月訓練，後一個月驗證和測試）。純社交互動數據，未計入訂單簿容量或滑點，容量假設未驗證。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations:** 訓練時間增加；正樣本增強僅緩解不平衡，單獨移除影響有限；時間採樣策略在特定高難度任務上略遜於專門設計的Historical策略。
**6.2 推斷的隱含假設:** Regime依賴極強（僅適用於Meme股狂熱期，常態市場社交圖稀疏且無金融意義）；容量假設未驗證（高頻5分鐘圖嵌入的計算開銷與實盤延遲）；數據泄漏風險（正樣本增強引入未來邊，若批次劃分邊界處理不當易洩漏）；倖存者偏差（僅選取GME/AMC/BB等已爆發標的）。
