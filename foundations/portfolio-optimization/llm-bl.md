# LLM-BL 解構（LLM-BL）

> **發布**：2025-04-24 · （無 venue）
> **QuantML 導讀**：[融合LLM观点的BL模型](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247490164&idx=1&sn=c0d59c459086d6592c788380db653878&chksm=ce7e7d6af909f47c79623d45235153d3db8fb6c74b1b8fd3fbfbc6e269318472d22bbf868134#rd)
> **核心定位**：落點於「生成式大模型 × 組合執行優化」軸。解了傳統 Black-Litterman 模型中「觀點來源高度依賴主觀判斷」且難以系統化、自動化擴展的工程痛點，將觀點生成轉為數據驅動的閉環。

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** 將 LLM 對個股未來兩週平均日收益的多次預測均值與方差，直接映射為 Black-Litterman 模型的觀點向量與置信度矩陣，實現全自動觀點生成與雙週調倉組合優化。核心 trick 在於用 LLM 輸出方差替代傳統主觀置信度設定，將模型「不確定性」轉化為優化器的風險懲罰項。這對「組合執行優化」軸★ 意味著觀點生成從離散人工標註轉向連續數據驅動，但模型間表現差異極大（如 BLM-Llama CAGR 0.6731 顯著勝出，MVO CAGR -0.0189 失效），顯示 LLM 的「預測樂觀性與信心穩定性」是 alpha 來源的關鍵閾值。

**X-Ray.** 放回五軸 Pareto，本法將「觀點生成」從傳統因子挖掘抽離，轉嫁至 LLM 的上下文推理能力。它解了 BL 模型長期面臨的「觀點矩陣 P 與 Ω 設定缺乏客觀錨點」的舊坑，用 LLM 多次採樣的方差直接對角化 Ω，實現了置信度的動態標定。然而，其打不開的 envelope 在於：LLM 對純量價時間序列的歸納偏誤極高，且優化器對觀點均值極度敏感。回測僅覆蓋 2024-06 至 2025-02 的單邊/震盪市，未計交易成本與滑點，雙週調倉在實盤中可能因 LLM 推理延遲產生執行摩擦。對量化讀者而言，此法並非預測模型，而是「觀點過濾與風險預算分配器」；其價值不在於 LLM 預測多準，而在於方差如何平滑 MVO 的極端權重。若無法解決 LLM 輸出分佈的 regime 漂移，該框架僅能作為輔助權重錨，而非獨立 alpha 源。

## §1 · 架構 / Core Mechanism
**1.1 三大改動 vs 前作**
| 維度 | 傳統 BL / MVO | LLM-BL (本法) | 改動意圖 |
|---|---|---|---|
| 觀點來源 (q) | 主觀判斷/宏觀模型/分析師一致預期 | LLM 多次查詢均值 (10次) | 自動化、數據驅動、消除人為錨定 |
| 置信度矩陣 (Ω) | 經驗設定/固定比例/主觀賦權 | LLM 預測結果方差 (對角矩陣) | 將模型不確定性轉化為優化器風險懲罰 |
| 調倉頻率/數據流 | 月頻/季頻，依賴靜態截面 | 雙週頻，依賴滾動兩週日頻回報+元數據 | 提升對短期動量/反轉的適應性，但增加推理成本 |

**1.2 ⚡ Eureka 一句話 trick + 直覺**
Trick：用 LLM 對同一 prompt 的多次輸出方差直接填充 Black-Litterman 的 Ω 對角線。
直覺：LLM 越「猶豫」（方差大），優化器給該觀點的權重越低；LLM 越「堅定」（方差小且均值極端），權重越高。這本質上是用生成模型的內在熵值做動態風險預算。

**1.3 信息流 ASCII 圖**
```
[歷史兩週日頻回報 + 公司元數據]
          ↓
[LLM Prompt (10次採樣)] → 輸出序列 {r1, r2, ..., r10}
          ↓
[統計聚合] → μ_view = mean({r}), σ²_view = var({r})
          ↓
[BL 映射] → q = μ_view, Ω = diag(σ²_view), P = I
          ↓
[後驗回報計算] → π_prior (CAPM) + BL 調整 → μ_posterior
          ↓
[組合優化] → w* = argmax(μ_posterior, Σ)
          ↓
[雙週調倉執行]
```

## §2 · 數學層
📌 **Napkin Formula**：
`μ_posterior = [ (τΣ)^-1 + P'Ω^-1 P ]^-1 [ (τΣ)^-1 π + P'Ω^-1 q ]`
複雜度：LLM 推理 O(N_stocks × N_queries × LLM_params)，BL 閉式解 O(N_stocks^3)（矩陣求逆），整體瓶頸在 LLM 吞吐量。
直覺：公式將先驗均衡回報 π 與 LLM 觀點 q 進行貝葉斯加權平均。權重由 Ω（LLM 方差）與 τ（先驗不確定性）共同決定。方差越大，Ω^-1 越小，觀點 q 對 μ_posterior 的拉力越弱，組合權重越趨近先驗均衡。
Loss/訓練細節：無傳統梯度下降訓練。LLM 為指令調優凍結權重，僅做 In-context 推理；BL 參數 τ 與 Σ 估計窗長導讀未披露。

## §3 · 數據層
資料規模/頻率/市場/時段：標普500市值前50成分股，日頻回報數據，2024-06至2025-02（8個月）。
怎麼來：Yahoo Finance 歷史價格 + 公司元數據（代碼、名稱、GICS行業/子行業）。
樣本外與容量假設：嚴格樣本外（模型知識截止日早於數據期），但僅覆蓋單一市場（美股）與單一市值段（大盤）。容量假設受限於 LLM 推理延遲與雙週調倉頻率，實盤容量約為前50大盤股流動性上限，未計交易成本與滑點。

## §5 · 評測 / Benchmark
| 數據集/市場 | Metric | 前SOTA | 本方法 | Δ |
|---|---|---|---|---|
| 標普500前50/美股 | CAGR | -0.0189 (MVO) | 0.6731 | +0.6920 |
| 標普500前50/美股 | Sharpe | 2.5075 (EW) | 未披露 | 未披露 |
| 標普500前50/美股 | Volatility | 0.0074 (EW) | 未披露 | 未披露 |

**解讀**：Δ 的真 capability 在於 LLM 觀點成功規避了傳統 MVO 對參數誤差的極度敏感（MVO Sharpe 僅 0.0087 且 CAGR 負值），證明「方差錨定置信度」能有效收斂極端權重。但 EW 的 Sharpe 高達 2.5075 且波動僅 0.0074，暗示回測期可能處於低波動/趨勢市，簡單分散化已極優。BLM-Llama 的 CAGR 優勢可能部分來自其觀點分佈的「極端異常值過濾機制」，而非純粹的預測準確率。未計交易成本與雙週調倉摩擦，實盤 Δ 將顯著縮水。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations**：未納入具有系統超參數調整的驗證期；未整合金融文本信息（如新聞/財報）；不同 LLM 表現差異大（Gemma/GPT 負收益），顯示模型選擇對結果影響極大。
**6.2 推斷的隱含假設**：
- Regime 依賴：雙週調倉+日頻輸入假設市場存在短期可捕捉的動量/反轉結構，若進入高頻噪聲或結構性斷裂期，LLM 的上下文歸納將失效。
- 容量/成本：前50大盤股假設流動性充足，未計雙週調倉的衝擊成本與 LLM API 延遲導致的執行滑點。
- 數據泄漏：提示詞僅給過去兩週數據，但 LLM 訓練數據可能已內化歷史宏觀模式，存在隱性數據泄漏風險。
- 方差穩定性：10次查詢的方差作為 Ω 對角線，樣本量極小，可能無法穩定估計真實預測不確定性，易受 LLM 隨機種子/溫度參數干擾。

## §7 · 對比 & 面試 Tip
| 同軸對手 | 關鍵差異軸 | Open? | Status |
|---|---|---|---|
| 傳統 BL (主觀觀點) | 觀點來源與置信度設定方式 | 是 | 工業界標準，但擴展性差 |
| 純 LLM 預測+等權重 | 是否引入貝葉斯風險預算分配 | 是 | 實驗性，易受預測誤差放大 |
| RL/強化學習組合優化 | 優化目標與訓練範式（序列決策 vs 閉式解） | 部分 | 高計算成本，需大量交互環境 |

🎤 **Interview Tip**
正確答：「本法核心不在於 LLM 預測多準，而在於用 LLM 輸出方差動態標定 Black-Litterman 的置信度矩陣 Ω，將生成模型的不確定性轉化為優化器的風險懲罰項，從而平滑傳統 MVO 的極端權重問題。」
錯答：「LLM 能準確預測股票未來兩週收益，所以我們直接用 LLM 的預測值做加權，效果比等權重好。」（忽略了 BL 框架的貝葉斯調整本質與方差錨定的工程價值）

**7.1 可證偽預測帶日期**：若實盤納入交易成本後，組合表現未能維持導讀給出的 CAGR 0.6731 水平，或 LLM 輸出方差與實際實現波動率脫鉤，則「方差錨定置信度」假設失效。

## §8 · For the Reader
- **因子研究員**：關注 LLM 輸出分佈的橫截面偏度（如 Llama 的負向極值過濾），可嘗試將該分佈特徵轉化為動量/反轉因子，而非直接用於權重優化。
- **高頻執行**：雙週調倉對執行壓力小，但需警惕 LLM API 限流與推理延遲導致的信號衰減；建議將 LLM 推理與組合優化解耦，採用異步預計算架構。
- **組合配置/LLM-agent**：本法是典型的「LLM as Oracle」範式。若需升級為 Agent，應加入反饋迴路（如將實盤誤差回傳調整 Prompt 或 Ω 縮放係數），並引入多模態數據修正純量價歸納偏誤。
- **研究學生**：可復現但需警惕過擬合。建議擴展 τ 參數掃描與 Ω 正則化，並對比不同 LLM 架構在相同 Prompt 下的方差穩定性。
