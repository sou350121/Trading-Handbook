> **核心定位**：將多模態股價預測的瓶頸從模型架構轉移至數據質量與因子可解釋性，以 LLM 序列提示構建結構化因子錨點，配合輕量級三層框架降低 A 股多模態研發門檻。

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 構建中國股市多模態數據集 CSMD 與輕量級回測框架 LightQuant。② 核心 trick 為基於 LLM 序列提示從財經新聞中提取可解釋金融因子，配合三層模塊化架構實現數據清洗、建模與回測一站式流程。③ 對「端到端表徵」與「人機協同可解釋」軸★：將黑盒文本轉化為結構化、可驗證的因子，兼顧模型表現與風控審查需求。④ 導讀未給完整量化結果，僅披露 StockNet 在 CSMD 300 預測 ACC 為 55.47，回測 ARR 為 0.1301。

**X-Ray.** 放回五軸 Pareto，LightQuant 本質是「工程降維」而非「算法突破」。它直擊傳統 NLP 因子黑盒化與回測框架陡峭的舊坑，以 LLM 序列提示實現軟性特徵工程，將文本映射顯式拆解為因子提取與價格預測兩階段，避開端到端嵌入的梯度不穩定。預測其打不開的 envelope 在於：LLM 提取的因子高度依賴新聞語境與提示詞設計，對盤中微結構、訂單簿異動或未公開內幕無響應；且日頻波段設定下，因子更新頻率與交易成本摩擦將稀釋理論收益。對量化讀者而言，此框架的價值不在於直接產出 Alpha，而在於提供一套標準化的多模態數據處理與原型驗證流水線，適合快速篩選文本驅動型假說，但實盤部署前必須補齊滑點模型、流動性約束與 LLM 調用成本測算。

## §1 · 架構 / Core Mechanism
**1.1 三大改動 vs 前作**
| 維度 | 傳統多模態/回測框架 (如 Qlib/StockNet 原生) | LightQuant + CSMD |
|---|---|---|
| 數據來源與對齊 | 英語市場為主，價格與文本時間戳常不同步 | 專為 A 股定制，證券時報新聞與 Baostock 價格嚴格時間對齊 |
| 因子提取機制 | 依賴預訓練語言模型隱空間表徵或簡單情感打分 | LLM 序列提示提取結構化、可解釋金融因子 |
| 框架架構 | 功能冗餘、學習曲線陡峭、模塊耦合度高 | 三層模塊化（數據/模型/評估），即插即用，內置可視化與日誌 |

**1.2 ⚡ Eureka 一句話 trick + 直覺**
Trick：用精心設計的序列知識提示詞引導 LLM 將非結構化財經新聞轉化為「企業盈利/宏觀影響/市場情緒」三維可解釋因子。
直覺：避開端到端文本嵌入的不可控性，以人類可審計的因子作為跨模態對齊的錨點，兼顧模型表現與合規/風控審查需求。

**1.3 信息流 ASCII 圖**
```
[新聞爬取 (STCN)] --> [LLM 序列提示] --> [結構化因子庫]
       |                                      |
       v                                      v
[價格數據 (Baostock)] --> [時間對齊/去噪] --> [多模態輸入矩陣]
                                             |
                                             v
[LightQuant 模型層 (LSTM/StockNet/HAN等)] --> [評估層 (ACC/MCC/ARR/SR/MDD/CR)]
                                             |
                                             v
[回測報告/可視化/日誌]
```

## §2 · 數學層
📌 **Napkin Formula**:
$F_{t} = \text{LLM}_{\theta}(\text{Prompt}_{seq}, \text{News}_{t})$
$Y_{t+1} = f_{\phi}(\text{Price}_{t}, F_{t})$
複雜度：LLM 推理 $O(N \cdot L)$，下游預測模型依架構而定（如 StockNet 為跨模態注意力 $O(T^2)$）。
直覺：將文本映射顯式拆解為因子提取與價格預測兩階段，避免梯度在長文本與高頻價格間直接傳播導致的不穩定。
Loss/訓練：導讀僅提及監控 Loss、Val ACC、Val MCC，未披露具體損失函數形式與正則化細節。

## §3 · 數據層
資料規模/頻率/市場/時段：覆蓋滬深300與上證50成分股，日頻波段。數據源自證券時報（新聞）與 Baostock（價格）。
怎麼來：自動化爬取管線 + 指數退避策略 + 結構化解析。
樣本外與容量假設：導讀未披露具體時間跨度與樣本量。假設為公開成分股，存在潛在幸存者偏差；日頻設定下容量假設較高，但需計入 LLM 調用成本與交易滑點。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations**：導讀未明確列出 limitations 章節，僅提及現有開源數據集缺乏中國市場覆蓋、原始文本噪聲高、研發門檻高。
**6.2 推斷的隱含假設**：
- Regime 依賴：LLM 因子提取高度依賴新聞語境與提示詞設計，在政策急轉或情緒極端行情下可能失效或產生滯後。
- 容量/成本：日頻波段假設流動性充足，但 LLM 逐條推理成本與 API 限流將限制股票池擴張與因子更新頻率。
- 數據泄漏/幸存者：成分股數據天然包含幸存者偏差；新聞時間戳與收盤價對齊若未精確到分鐘級，易引入前瞻偏差。
- 評估指標局限：ACC/MCC 未區分牛熊週期，回測未披露換手率與持倉集中度，無法判斷策略是否依賴極端集中押注。

## §8 · For the Reader
- **因子研究員**：可直接將 CSMD 的 LLM 提取因子接入現有因子庫，驗證文本驅動因子的 IC/IR 衰減曲線，注意提示詞穩定性與跨市場遷移能力。
- **高頻執行/組合配置**：日頻波段設定與低 MDD 指標不匹配高頻邏輯；需將框架輸出對接至組合優化器，並加入流動性與交易成本約束。
- **LLM-agent/RL 策略**：可將 LightQuant 的評估層作為 Reward 環境，訓練 RL Agent 動態調整 LLM 提示詞權重或因子加權策略。
- **研究學生**：框架極適合作為多模態量化入門的標準化流水線，但需自行補齊數據對齊細節與回測合規性檢查。
