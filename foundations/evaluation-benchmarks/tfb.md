# TFB 解構（TFB）

> **發布**：2024-10-01 · （無 venue）
> **QuantML 導讀**：[TFB：全面与公平的时间序列预测方法基准测试](https://mp.weixin.qq.com/s?__biz=Mzg2MzAwNzM0NQ==&mid=2247486888&idx=1&sn=53c296e3586dcdd37e37d9a97a945fa1&chksm=ce7e6ab6f909e3a0e3f38effb99b0a2e66b2bd4753d3c30f1f6212cfc41171d0965d144f62e8#rd)
> **核心定位**：落點於「跨周期量價表格監督回歸」軸，解決了 TSF 領域長期存在的「數據集碎片化、方法比較偏見、評估流程不一致」三大工程斷層，提供跨統計/ML/DL 的標準化公平評測基準。

**Status:** v0.5 — 基於 QuantML 導讀 + 原論文（如有）。benchmark 細節待升 v1。
**TL;DR:** ① 構建涵蓋10大領域、8000+單變量與25個多變量時間序列的統一基準測試（TFB）。② 核心 trick 在於以PCA特徵描述篩選數據集，並設計「數據-方法-評估-報告」四層標準化流水線，強制消除領域與方法偏見。③ 對「因子挖掘/監督回歸」軸★的關鍵意義在於：將Alpha回測從「手工作坊」升級為「工業級對照實驗」，明確量化線性模型與Transformer在不同序列特徵下的真實能力邊界。④ 關鍵實證：系統評估35種方法，證實通道獨立性假設在多變量預測中並非絕對優勢，且線性基線在特定特徵數據集上仍具競爭力（具體數值未披露）。

**X-Ray.** TFB 本質上不是新模型，而是量化研究基礎設施的「標準化協議」。在五軸 Pareto 中，它強行將「監督回歸」與「因子挖掘」從黑盒實驗拉回可解釋的對照實驗框架。它解了三個舊工程坑：① 數據集選擇偏見（以往 benchmark 多集中於少數公開數據，TFB 用 PCA 覆蓋趨勢、季節性、平穩性等特徵）；② 評估流程不一致（固定 vs 滾動預測、超參數搜索策略不透明）；③ 方法比較偏見（過度偏向 DL，忽略統計/ML 基線）。預測其打不開的 envelope：TFB 仍停留在「純預測精度」層面，未觸及量化實盤最關鍵的「交易成本、滑點、容量約束與風險預算」，因此其結論不能直接映射為 PnL。對量化讀者的意義在於：它提供了一套可複用的特徵-方法映射指南，能幫助研究員在開發新 Alpha 前，快速定位數據的統計屬性，避免盲目上 Transformer 或過度工程化。

## §1 · 架構 / Core Mechanism
**1.1 三大改動 vs 前作**
| 維度 | 前作 (Libra/BasicTS/Monash) | TFB |
|---|---|---|
| 數據覆蓋 | 碎片化/單一領域/缺乏特徵描述 | 10領域/8000+序列/PCA五維特徵分佈 |
| 方法範圍 | 偏重 DL 或 ML，忽略統計基線 | 統計/ML/DL 全覆蓋 (35種) |
| 評估流程 | 手動/不一致/超參不透明 | 四層自動化流水線 (數據-方法-評估-報告) |

**1.2 ⚡ Eureka 一句話 trick + 直覺**
「以特徵分佈反推方法選擇」——用 PCA 量化序列的趨勢/季節性/平穩性/偏移/轉換，強制要求 benchmark 覆蓋全特徵空間，從而消除「挑數據刷榜」的偏見。直覺：不拼模型深度，拼實驗設計的對照嚴謹度。

**1.3 信息流 ASCII 圖**
```
[Data Layer] → 統一格式/PCA特徵驗證/存儲
       ↓
[Method Layer] → 統計/ML/DL 35種方法集成/超參協議
       ↓
[Eval Layer] → 固定/滾動策略切換/MAE-MSE-SMAPE-RMSE 計算
       ↓
[Report Layer] → 日誌追蹤/可視化/能力邊界映射指南
```

## §2 · 數學層
📌 **Napkin Formula**：$\mathcal{L}_{eval} = \{MAE, MSE, SMAPE, RMSE\}$ (統一計算口徑) | 複雜度: $O(N \cdot L \cdot H \cdot M)$ (N:序列數, L:長度, H:預測步長, M:方法數)，流水線自動化開銷 TBD。
**直覺**：TFB 不發明新數學，它發明的是「對照實驗的約束條件」。通過固定超參搜索空間與評估策略，將方法性能差異歸因於模型架構而非調參運氣。
**Loss/訓練細節**：沿用各方法原始論文設定，TFB 層面不定義統一 Loss，僅統一評估指標計算口徑與超參搜索協議。

## §3 · 數據層
- **資料規模/頻率/市場/時段**：10大領域（交通、電力、能源、環境、自然、經濟、股票、銀行、健康、網絡）。單變量 8068 個，多變量 25 個。具體頻率/時段/市場範圍未披露。
- **怎麼來**：開源數據集篩選 + PCA 特徵驗證，確保反映真實世界複雜性。
- **樣本外與容量假設**：採用標準時間序列劃分（訓練/驗證/測試），具體劃分比例與樣本外窗口未披露。假設數據具備時間序列標準的平穩性/特徵穩定性（經 PCA 驗證）。

## §5 · 評測 / Benchmark
| 數據集/市場 | Metric(IR/Sharpe/AR/MDD) | 前SOTA | 本方法 | Δ |
|---|---|---|---|---|
| 10大領域/8068 UTSF + 25 MTSF | 未披露（純預測誤差MAE/MSE/SMAPE/RMSE） | 未披露 | 未披露 | 未披露 |

**解讀**：Δ 反映的是純預測精度差異，非交易收益。DL 在複雜特徵數據上通常領先，但線性模型在趨勢明確數據上表現穩健。部分 Δ 可能源於超參數搜索充分度差異，非模型絕對能力。無成本/風險調整，結論僅限於預測層面。

## §6 · 失效與隱含假設
**6.1 論文自述 limitations**：未明確列出，但導讀暗示聚焦於預測精度，未涵蓋實盤摩擦與風險預算。
**6.2 推斷的隱含假設**：
- **Regime 依賴**：PCA 特徵分佈假設歷史特徵穩定，未處理結構性斷點（Structural Breaks）。
- **容量/成本**：純預測誤差 vs 交易執行，忽略衝擊成本與流動性約束。
- **數據泄漏**：滾動預測若未嚴格隔離未來信息或特徵工程包含未來視角，可能引入偏見。
- **Survivorship**：開源數據集通常已過濾退市/失效標的，存在倖存者偏差。

🎤 **Interview Tip**：
- **正確答**：TFB 是基礎設施/協議，解決比較偏見與流程不一致，結論需映射至實盤成本框架；其價值在於提供「數據特徵-方法選擇」的對照指南。
- **錯答**：TFB 是一個新的預測模型/Transformer 變體，能直接提升 Sharpe 或替代現有 Alpha 挖掘流程。

**7.1 可證偽預測帶日期**：若 2025-Q2 前無公開 GitHub 倉庫與 Docker 化部署腳本，則其「四層流水線」僅為概念驗證，無法支撐工業級複現。

## §8 · For the Reader
- **因子研究員**：用 PCA 特徵描述快速分類新 Alpha 數據，避免在平穩序列上強上複雜 DL，優先驗證線性基線。
- **高頻執行**：忽略純 MAE/MSE 提升，將 TFB 的 Eval Layer 替換為 Impact Cost & Slippage 模擬器，重構 Reward 函數。
- **組合配置**：將 TFB 的「通道獨立性 vs 依賴性」結論應用於多資產組合的協方差矩陣估計，而非單序列預測。
- **LLM-agent/RL**：將 TFB 流水線作為 Reward 計算的標準化環境，但需重寫 Eval Layer 以支持離線 RL 的 OPE 指標。
- **研究學生**：直接繼承 Data Layer 與 Report Layer 代碼結構，專注於 Method Layer 的架構創新，省去數據清洗與評估腳本開發。
